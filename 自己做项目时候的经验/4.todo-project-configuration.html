Todo 项目配置修改（适配服务器环境）

项目部署到服务器后，需修改后端数据库连接和前端接口请求地址，确保前后端能正常通信。

一、修改后端数据库连接（db.js）

db.js 是数据库连接配置文件，需填写服务器 MySQL 的真实信息。

# 1. 进入后端目录（确保项目已上传到服务器 /root/todo-project）
cd /root/todo-project/backend

# 2. 用 nano 编辑器打开 db.js（Ubuntu 自带，简单易用）
nano db.js

db.js 配置示例（替换为你的信息）

// 引入 mysql2 模块
const mysql = require('mysql2');

// 创建数据库连接池
const pool = mysql.createPool({
  host: 'localhost',    // 本地 MySQL，默认 localhost
  user: 'root',         // MySQL 用户名（默认 root）
  password: '你的MySQL密码', // 之前设置的 MySQL root 密码
  database: 'todo_db',  // 项目数据库名（已创建的 todo_db）
  waitForConnections: true,
  connectionLimit: 10,
  queueLimit: 0
});

// 导出连接池（供 server.js 调用）
module.exports = pool.promise();
编辑完成后，按 Ctrl+O 保存，按 Ctrl+X 退出 nano 编辑器。

二、修改前端接口请求地址（script.js）

前端需通过服务器公网 IP 调用后端 API，需修改 script.js 中的接口前缀。

# 1. 进入前端目录
cd /root/todo-project/frontend

# 2. 用 nano 编辑器打开 script.js
nano script.js

script.js 关键修改点

找到所有接口请求地址，将 localhost:3000 替换为你的服务器公网 IP（无需加端口，Nginx 会反向代理）：

// 原接口地址（本地开发环境）
const baseUrl = 'http://localhost:3000/api';

// 修改后（服务器环境，替换为你的公网 IP）
const baseUrl = 'http://你的服务器公网IP/api';

// 示例：登录接口请求（修改后）
async function login(username, password) {
  const response = await fetch(`${baseUrl}/login`, {
    method: 'POST',
    headers: { 'Content-Type': 'application/json' },
    body: JSON.stringify({ username, password })
  });
  return response.json();
}
注意：所有接口（登录、注册、待办增删改查）都需使用统一的 baseUrl，避免部分接口失效。
