Todo 项目结构与文件布局

Node.js+Express+MySQL Todo 项目采用前后端分离结构，清晰划分前端静态资源和后端接口逻辑，便于维护和扩展。

项目整体结构

todo-project/
├─ frontend/  # 前端静态文件（用户交互界面）
│  ├─ login.html    # 登录页面
│  ├─ register.html # 注册页面
│  ├─ todolist.html # 待办事项主页面
│  ├─ style.css     # 全局样式文件
│  └─ script.js     # 前端交互逻辑（接口请求、DOM操作）
└─ backend/   # 后端服务文件（接口+数据库交互）
   ├─ db.js    # 数据库连接配置（连接 MySQL）
   └─ server.js # 后端核心逻辑（Express 接口、路由、权限验证）

各目录/文件说明

1. frontend/ 前端目录

  • login.html：用户登录界面，包含账号密码输入框和登录按钮
  • register.html：用户注册界面，实现用户名/密码注册功能
  • todolist.html：核心功能页面，展示、添加、编辑、删除待办事项
  • style.css：统一样式文件，美化所有页面布局和元素
  • script.js：前端逻辑核心，负责调用后端 API（登录、注册、待办操作）、页面状态更新

2. backend/ 后端目录

  • db.js：数据库连接配置文件，定义 MySQL 主机、端口、用户名、密码、数据库名，供 server.js 调用
  • server.js：后端入口文件，基于 Express 框架实现：
    • 路由定义（/api/login、/api/register、/api/todos 等接口）
    • 请求处理（接收前端参数、逻辑判断）
    • 数据库操作（调用 db.js 连接，执行增删改查）
    • 权限验证（如 JWT 令牌验证，保护待办事项接口）
注意：项目部署时，前端文件通过 Nginx 提供静态资源访问，后端接口通过 Nginx 反向代理转发到 Node.js 服务端口（默认 3000）。
