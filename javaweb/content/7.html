第6章：JDBC 数据库核心（小白完整版·Maven+MySQL8）

6.0 前置核心：JDBC 是干嘛的？

术语：JDBC（Java DataBase Connectivity）Java 数据库连接
👉 大白话：Java 官方提供的、用来连接 MySQL 等数据库的工具包。

之前的问题：
我们之前的登录、数据都是写死在代码里的，重启服务器就消失。
JDBC 的作用：
让 Java 代码连接 MySQL，把数据存到数据库里、从数据库里读数据，实现永久保存。

💡 最通俗类比：
📌 JDBC = Java 和 MySQL 之间的翻译官+桥梁
📌 Java 说 Java 的话，MySQL 说 MySQL 的话，JDBC 负责传话

本章前提：你电脑已安装 MySQL8.0+，有可视化工具（Navicat/SQLyog）。

6.1 Maven 依赖配置（MySQL8 专用驱动）

JDBC 本身是 Java 自带的，但MySQL 驱动需要手动引入，在 pom.xml 添加：

MySQL8.0+ 驱动 Maven 坐标（小白直接复制） 
<!-- MySQL8 数据库驱动 -->
<dependency>
    <groupId>com.mysql</groupId>
    <artifactId>mysql-connector-j</artifactId>
    <version>8.0.33</version>
</dependency>
❗ 小白必坑：
1. MySQL5 用 mysql-connector-java，MySQL8 用 mysql-connector-j
2. 版本必须对应你的 MySQL 版本（8.0.x 通用）

6.2 JDBC 固定 7 步（背会这7步，所有数据库操作都会）

不管是查询、添加、修改、删除，全程都按这7步走，一步都不能少！

JDBC 固定7步（小白必背）
  1. 导入驱动包（Maven 已导入）
  2. 注册驱动（MySQL8 自动注册，不用写代码）
  3. 获取数据库连接（和 MySQL 打通通道）
  4. 编写要执行的 SQL 语句
  5. 获取 SQL 执行对象（PreparedStatement，99%项目用这个）
  6. 执行 SQL，处理结果
  7. 关闭资源（先开后关，逆序关闭）

6.2.1 核心对象（大白话解释，只记常用4个）

对象名 作用（大白话） 常用度
DriverManager 驱动管理类，获取数据库连接 ⭐⭐⭐⭐⭐
Connection（连接对象） Java 和 MySQL 的通道 ⭐⭐⭐⭐⭐
PreparedStatement（预编译对象） 执行 SQL 的工具，安全防注入 ⭐⭐⭐⭐⭐
ResultSet（结果集） 查询时，封装 MySQL 返回的数据 ⭐⭐⭐⭐⭐
✅ 已删除极少用对象：Statement（不安全，企业99%不用）

6.4 实战1：查询数据库中所有用户（最常用）

需求：用 JDBC 从 user 表中读取所有用户数据，打印到控制台。

完整代码（JdbcSelect.java）

package com.jdbc;

import java.sql.Connection;
import java.sql.DriverManager;
import java.sql.PreparedStatement;
import java.sql.ResultSet;

// JDBC 查询所有用户
public class JdbcSelect {
    public static void main(String[] args) throws Exception {
        // ================= 第1-2步：驱动已自动处理 =================

        // ================= 第3步：获取数据库连接（固定4个参数） =================
        // 1. 连接地址（MySQL8 必须加时区、编码）
        String url = "jdbc:mysql://localhost:3306/javaweb_test?serverTimezone=UTC&useSSL=false&characterEncoding=utf8";
        // 2. MySQL 用户名（默认root）
        String user = "root";
        // 3. MySQL 密码（你自己的密码）
        String password = "123456";

        // 获取连接（Connection：通道）
        Connection conn = DriverManager.getConnection(url, user, password);

        // ================= 第4步：编写SQL =================
        String sql = "SELECT * FROM user";

        // ================= 第5步：获取执行对象 =================
        PreparedStatement pstmt = conn.prepareStatement(sql);

        // ================= 第6步：执行SQL，处理结果 =================
        // executeQuery()：执行查询，返回结果集 ResultSet
        ResultSet rs = pstmt.executeQuery();

        // 遍历结果集（rs.next()：判断有没有下一条数据）
        while (rs.next()) {
            // 根据列名获取数据
            int id = rs.getInt("id");
            String username = rs.getString("username");
            String pwd = rs.getString("password");
            int age = rs.getInt("age");
            String address = rs.getString("address");

            // 打印
            System.out.println(id+"-"+username+"-"+pwd+"-"+age+"-"+address);
        }

        // ================= 第7步：关闭资源（逆序关：结果集→执行对象→连接） =================
        rs.close();
        pstmt.close();
        conn.close();
    }
}
💡 小白必改：
1. password 改成你自己的 MySQL 密码
2. 数据库名 javaweb_test 要和你建的一致

6.5 实战2：向数据库添加用户（增，常用）

需求：往 user 表中添加一条新用户数据。

完整代码（JdbcInsert.java）

package com.jdbc;

import java.sql.Connection;
import java.sql.DriverManager;
import java.sql.PreparedStatement;

// JDBC 添加用户
public class JdbcInsert {
    public static void main(String[] args) throws Exception {
        // 1. 获取连接
        String url = "jdbc:mysql://localhost:3306/javaweb_test?serverTimezone=UTC&useSSL=false&characterEncoding=utf8";
        String user = "root";
        String password = "123456";
        Connection conn = DriverManager.getConnection(url, user, password);

        // 2. 编写SQL（? 是占位符，后面赋值，防SQL注入）
        String sql = "INSERT INTO user(username,password,age,address) VALUES(?,?,?,?)";

        // 3. 获取执行对象
        PreparedStatement pstmt = conn.prepareStatement(sql);

        // 4. 给占位符赋值（下标从1开始）
        pstmt.setString(1, "zhangsan"); // 第1个?：username
        pstmt.setString(2, "888888");  // 第2个?：password
        pstmt.setInt(3, 25);          // 第3个?：age
        pstmt.setString(4, "广州");    // 第4个?：address

        // 5. 执行增删改：executeUpdate()，返回受影响行数
        int row = pstmt.executeUpdate();
        System.out.println("添加了 " + row + " 条数据");

        // 6. 关闭资源
        pstmt.close();
        conn.close();
    }
}
大白话：? 是占位符，相当于填空题，后面用 setXxx() 填内容，绝对安全。

6.7 JDBC 工具类（简化重复代码，90%项目必用）

问题：每次都写连接、关闭代码，太重复。
解决：写一个工具类（JDBCUtil），把连接、关闭封装起来，直接调用。

JDBC 工具类代码（JDBCUtil.java）

package com.util;

import java.sql.Connection;
import java.sql.DriverManager;
import java.sql.ResultSet;
import java.sql.PreparedStatement;

// JDBC 工具类：封装连接、关闭
public class JDBCUtil {
    // 固定4个参数（改成你自己的）
    private static final String URL = "jdbc:mysql://localhost:3306/javaweb_test?serverTimezone=UTC&useSSL=false&characterEncoding=utf8";
    private static final String USER = "root";
    private static final String PWD = "123456";

    // 1. 获取连接
    public static Connection getConn() throws Exception {
        return DriverManager.getConnection(URL, USER, PWD);
    }

    // 2. 关闭资源（查询：关3个）
    public static void close(Connection conn, PreparedStatement pstmt, ResultSet rs){
        try {
            if(rs != null) rs.close();
            if(pstmt != null) pstmt.close();
            if(conn != null) conn.close();
        } catch (Exception e) {
            e.printStackTrace();
        }
    }

    // 3. 关闭资源（增删改：关2个）
    public static void close(Connection conn, PreparedStatement pstmt){
        close(conn, pstmt, null);
    }
}

工具类使用（简化后的查询）

package com.jdbc;

import com.util.JDBCUtil;
import java.sql.Connection;
import java.sql.PreparedStatement;
import java.sql.ResultSet;

// 使用工具类查询
public class JdbcUtilTest {
    public static void main(String[] args) {
        Connection conn = null;
        PreparedStatement pstmt = null;
        ResultSet rs = null;

        try {
            // 1. 获取连接（工具类）
            conn = JDBCUtil.getConn();
            String sql = "SELECT * FROM user";
            pstmt = conn.prepareStatement(sql);
            rs = pstmt.executeQuery();

            while (rs.next()) {
                System.out.println(rs.getString("username"));
            }
        } catch (Exception e) {
            e.printStackTrace();
        } finally {
            // 2. 关闭资源（工具类）
            JDBCUtil.close(conn, pstmt, rs);
        }
    }
}

6.8 综合实战：真实登录（Servlet+JDBC+MySQL，项目标配）

需求：
1. 前端输入用户名、密码
2. Servlet 接收参数
3. JDBC 查询 MySQL 中的用户
4. 匹配成功→登录成功，失败→提示错误

LoginServlet 完整代码

package com.servlet;

import com.util.JDBCUtil;
import jakarta.servlet.annotation.WebServlet;
import jakarta.servlet.http.HttpServlet;
import jakarta.servlet.http.HttpServletRequest;
import jakarta.servlet.http.HttpServletResponse;
import jakarta.servlet.http.HttpSession;
import java.sql.Connection;
import java.sql.PreparedStatement;
import java.sql.ResultSet;

@WebServlet("/login")
public class LoginServlet extends HttpServlet {
    @Override
    protected void doPost(HttpServletRequest request, HttpServletResponse response) throws Exception {
        // 1. 编码
        request.setCharacterEncoding("UTF-8");
        response.setContentType("text/html;charset=UTF-8");

        // 2. 获取前端参数
        String username = request.getParameter("username");
        String password = request.getParameter("password");

        // 3. JDBC 查询用户
        Connection conn = null;
        PreparedStatement pstmt = null;
        ResultSet rs = null;
        boolean isLogin = false;

        try {
            conn = JDBCUtil.getConn();
            String sql = "SELECT * FROM user WHERE username=? AND password=?";
            pstmt = conn.prepareStatement(sql);
            // 赋值
            pstmt.setString(1, username);
            pstmt.setString(2, password);
            rs = pstmt.executeQuery();

            // 如果有结果，说明登录成功
            if (rs.next()) {
                isLogin = true;
            }
        } finally {
            JDBCUtil.close(conn, pstmt, rs);
        }

        // 4. 处理结果
        HttpSession session = request.getSession();
        if (isLogin) {
            // 登录成功：存Session
            session.setAttribute("loginUser", username);
            response.sendRedirect("/home.jsp");
        } else {
            response.getWriter().write("用户名或密码错误！");
        }
    }
}

6.9 本章终极总结（小白记住这些就够）

🍉 JDBC 核心

  • JDBC = Java 连接 MySQL 的桥梁
  • 固定7步：导包→连接→写SQL→执行对象→执行→处理→关闭
  • MySQL8 驱动：mysql-connector-j

🔐 核心对象（必记）

  • Connection：数据库通道
  • PreparedStatement：安全执行SQL（防注入）
  • ResultSet：查询结果集

⚡ 执行方法（必记）

  • executeQuery()：查询，返回 ResultSet
  • executeUpdate()：增删改，返回受影响行数

📝 企业套路

  1. 用 JDBC 工具类 简化连接、关闭
  2. 全程用 PreparedStatement，不用 Statement
  3. Servlet+JDBC 实现登录、增删改查
✅ 本章只删除以下极少用内容：
1. Statement 普通执行对象（不安全）
2. JDBC 事务复杂配置（只留基础自动提交）
3. 数据库连接池（进阶内容，小白先学原生）
4. 元数据、批处理、存储过程调用（企业极少用）
