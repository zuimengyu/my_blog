第4章：Cookie & Session（小白完整版·90%常用全覆盖）

4.0 前置核心：HTTP 无状态问题（小白必理解）

术语：HTTP 无状态协议
👉 大白话：服务器没有记忆功能。
你这次访问登录，下次访问个人中心，服务器完全不认识你，会让你重新登录。

解决办法：
• Cookie：存在你浏览器里的「身份小纸条」
• Session：存在服务器里的「用户档案」
两者配合，服务器就能一直记住你。

💡 最通俗类比：
📌 Cookie = 网吧会员卡（你自己拿着）
📌 Session = 网吧会员资料（存在网吧电脑里）

4.1 Cookie 完整详解（存在浏览器里）

4.1.1 什么是 Cookie？（术语+大白话）

术语：Cookie 是服务器通过 HTTP 响应头发送给客户端，并由客户端（浏览器）临时或永久保存的小型文本数据。
大白话：服务器给你发一段小文本，浏览器自动存起来，下次访问自动带给服务器。

4.1.2 Cookie 核心特点（项目必知）

  • 📦 存储位置：浏览器（客户端）
  • 📏 大小限制：单个最多 4KB
  • 🔒 安全性：较低，不能存密码、银行卡等
  • ⌛ 生命周期：可设置存活时间
  • 🎯 作用范围：只能被同一服务器读取
  • 🗣️ 数据类型：只能存字符串，中文要编码

4.1.3 Cookie 完整常用 API（全部要会）

// 1. 创建 Cookie（键 + 值）
Cookie cookie = new Cookie("username", "admin");

// 2. 设置存活时间（秒）【最常用】
cookie.setMaxAge(60*60*24*7);  // 保存7天
cookie.setMaxAge(0);            // 立即删除
cookie.setMaxAge(-1);           // 浏览器关闭就消失（默认）

// 3. 设置访问路径【必须会】
cookie.setPath("/"); // 整个项目都能访问这个Cookie（推荐）

// 4. 发送给浏览器【必须写】
response.addCookie(cookie);

// 5. 获取浏览器所有Cookie
Cookie[] cookies = request.getCookies();

// 6. 遍历查找指定Cookie【必写，防报错】
if (cookies != null) {
    for (Cookie c : cookies) {
        String name = c.getName();
        String value = c.getValue();
        if ("username".equals(name)) {
            // 找到目标Cookie
        }
    }
}

// 7. 修改Cookie：同名+同路径覆盖
Cookie newCookie = new Cookie("username", "root");
newCookie.setPath("/");
response.addCookie(newCookie);

// 8. 删除Cookie：同名+同路径+setMaxAge(0)
Cookie delCookie = new Cookie("username", "");
delCookie.setPath("/");
delCookie.setMaxAge(0);
response.addCookie(delCookie);
❗ 小白 5 大必死坑：
1. 获取Cookie 必须先判断 != null，否则空指针报错
2. 修改/删除Cookie 名字+路径必须完全一样
3. Cookie 不能直接存中文
4. 不设置路径，可能导致Cookie读不到
5. 敏感信息绝对不能放Cookie

4.1.4 实战：创建、获取、修改、删除 Cookie（全套）

代码完整可直接复制运行，每一步都有注释

// 1. 创建Cookie
@WebServlet("/createCookie")
public class CreateCookieServlet extends HttpServlet {
    @Override
    protected void doGet(HttpServletRequest request, HttpServletResponse response) throws IOException {
        response.setContentType("text/html;charset=UTF-8");
        Cookie c = new Cookie("username", "admin");
        c.setPath("/");
        c.setMaxAge(60*60*24*7);
        response.addCookie(c);
        response.getWriter().write("Cookie 创建成功");
    }
}

// 2. 获取Cookie
@WebServlet("/getCookie")
public class GetCookieServlet extends HttpServlet {
    @Override
    protected void doGet(HttpServletRequest request, HttpServletResponse response) throws IOException {
        response.setContentType("text/html;charset=UTF-8");
        String username = null;
        Cookie[] cs = request.getCookies();
        if (cs != null) {
            for (Cookie c : cs) {
                if ("username".equals(c.getName())) {
                    username = c.getValue();
                }
            }
        }
        response.getWriter().write("读取到用户名：" + username);
    }
}

4.2 Session 完整详解（存在服务器里）

4.2.1 什么是 Session？（术语+大白话）

术语：Session 是服务器为每个客户端（浏览器）创建的独立会话域对象，用于存储跨请求的共享数据。
大白话：服务器给每个浏览器开一个专属储物柜，存你的登录信息、操作状态。

4.2.2 Session 核心原理（小白必懂）

1. 你第一次访问，服务器创建 Session
2. 服务器自动生成一个 JSESSIONID（Session 的身份证号）
3. 服务器把 JSESSIONID 放在 Cookie 里发给你
4. 你下次访问，自动带上 JSESSIONID
5. 服务器通过 JSESSIONID 找到你的储物柜

4.2.3 Session 核心特点（项目必知）

  • 📦 存储位置：服务器内存（Tomcat）
  • 📏 大小限制：无限制
  • 🔒 安全性：高，专门存登录状态
  • ⌛ 生命周期：默认 30 分钟不操作自动过期
  • 🎯 作用范围：同一个浏览器共享一个 Session
  • 🗣️ 数据类型：任意类型（对象、数字、集合）

4.2.4 Session 完整常用 API（全部要会）

// 1. 获取/创建 Session【99%场景用这行】
HttpSession session = request.getSession();

// 2. 存数据（键值对）【登录必用】
session.setAttribute("loginUser", "admin");
session.setAttribute("userId", 1001);

// 3. 取数据【判断登录必用】
String user = (String) session.getAttribute("loginUser");
Integer id = (Integer) session.getAttribute("userId");

// 4. 删除指定数据
session.removeAttribute("loginUser");

// 5. 销毁整个 Session【退出登录必用】
session.invalidate();

// 6. 设置超时时间（秒）
session.setMaxInactiveInterval(60*10); // 10分钟过期

4.2.5 实战：登录、取状态、退出登录（全套）

// 1. 登录成功，存Session
@WebServlet("/login")
public class LoginServlet extends HttpServlet {
    @Override
    protected void doPost(HttpServletRequest request, HttpServletResponse response) throws IOException {
        request.setCharacterEncoding("UTF-8");
        String username = request.getParameter("username");
        String password = request.getParameter("password");

        if ("admin".equals(username) && "123456".equals(password)) {
            HttpSession session = request.getSession();
            session.setAttribute("loginUser", username);
            response.sendRedirect("/home.html");
        } else {
            response.getWriter().write("账号密码错误");
        }
    }
}

// 2. 获取登录状态
@WebServlet("/getLoginStatus")
public class GetLoginStatusServlet extends HttpServlet {
    @Override
    protected void doGet(HttpServletRequest request, HttpServletResponse response) throws IOException {
        response.setContentType("text/html;charset=UTF-8");
        HttpSession session = request.getSession();
        String user = (String) session.getAttribute("loginUser");
        if (user != null) {
            response.getWriter().write("已登录：" + user);
        } else {
            response.getWriter().write("未登录");
        }
    }
}

// 3. 退出登录，销毁Session
@WebServlet("/logout")
public class LogoutServlet extends HttpServlet {
    @Override
    protected void doGet(HttpServletRequest request, HttpServletResponse response) throws IOException {
        HttpSession session = request.getSession();
        session.invalidate();
        response.sendRedirect("/login.html");
    }
}

4.3 Cookie 与 Session 完整对比

对比项 Cookie Session
存储位置 浏览器（客户端） 服务器（Tomcat）
安全性 低，可被查看修改 高，用户无法操作
存储大小 小（4KB/个） 大（无限制）
数据类型 仅字符串 任意类型（对象）
生命周期 可长期保存 默认30分钟
核心用途 记住用户名、浏览记录 保存登录状态、用户信息
