SpringBoot 深层复习与进阶（适配JDK24+Tomcat11）

一、SpringBoot 核心知识点深层复盘

SpringBoot是SSM的简化升级版，核心是「约定大于配置」，先吃透核心本质，再攻克企业级进阶难点：

核心特性 底层本质 解决的核心问题 企业级进阶方向
起步依赖（Starter） Maven依赖聚合+版本统一管理 手动导依赖、版本冲突、依赖缺失 自定义Starter、依赖瘦身
自动配置（AutoConfig） SPI机制+条件装配（@Conditional） 手动编写XML配置（Spring/MVC/MyBatis） 自动配置原理、自定义自动配置
内嵌服务器 Tomcat/Jetty/Undertow打包进JAR 手动配置外置Tomcat、部署WAR包 服务器优化、多端口配置
外部化配置 application.yml/properties统一配置 硬编码配置、多环境切换繁琐 多环境、配置加密、配置优先级
执行器（Actuator） 内置监控接口 项目运行状态不可见、问题排查难 自定义监控指标、监控告警
✅ 核心认知：SpringBoot不是新技术，而是对Spring/SSM的「封装+自动配置」，目的是提效 - 从「配置驱动」转向「约定驱动」。

二、自动配置原理（知其然知其所以然）

SpringBoot最核心的进阶点，理解后能解决90%的「配置不生效」「自定义配置」问题。

2.1 自动配置核心流程（SpringBoot启动时）

┌─────────────────────────────────────────────────┐
│ 1. 启动类加@SpringBootApplication                │
│ 2. 扫描META-INF/spring/org.springframework.boot.autoconfigure.AutoConfiguration.imports │
│ 3. 加载所有自动配置类（如TomcatAutoConfiguration）│
│ 4. 通过@Conditional注解判断是否生效              │
│ 5. 生效的配置类向IOC容器注入Bean                │
└─────────────────────────────────────────────────┘

2.2 核心注解拆解（@SpringBootApplication）

// @SpringBootApplication 是组合注解，核心包含3个注解：
@Target({ElementType.TYPE})
@Retention(RetentionPolicy.RUNTIME)
@Documented
@Inherited
@SpringBootConfiguration // = @Configuration，标记为配置类
@EnableAutoConfiguration // 开启自动配置（核心）
@ComponentScan(...) // 扫描组件（默认扫描启动类同级/子包）
public @interface SpringBootApplication {}

// 关键：@EnableAutoConfiguration 拆解
@Import(AutoConfigurationImportSelector.class) // 导入自动配置选择器
public @interface EnableAutoConfiguration {}

2.3 条件装配（@Conditional，自动配置的开关）

自动配置类是否生效，由@Conditional系列注解决定，企业级开发必懂：

注解 作用 实战场景
@ConditionalOnClass 类路径存在指定类才生效 有Tomcat依赖才加载Tomcat自动配置
@ConditionalOnMissingClass 类路径不存在指定类才生效 没有Redis依赖才加载默认缓存配置
@ConditionalOnBean IOC容器有指定Bean才生效 有DataSource才加载事务配置
@ConditionalOnMissingBean IOC容器无指定Bean才生效 用户没自定义Bean才加载默认Bean
@ConditionalOnProperty 配置文件有指定属性才生效 配置spring.redis.enabled=true才加载Redis配置

2.4 自定义自动配置（企业级进阶）

场景：封装通用功能为自定义Starter，供多个项目复用（如日志Starter、权限Starter）。

// 步骤1：编写自动配置类
package com.boot.starter.user.config;

import com.boot.starter.user.service.UserService;
import org.springframework.boot.autoconfigure.condition.ConditionalOnClass;
import org.springframework.boot.autoconfigure.condition.ConditionalOnMissingBean;
import org.springframework.boot.autoconfigure.condition.ConditionalOnProperty;
import org.springframework.context.annotation.Bean;
import org.springframework.context.annotation.Configuration;

@Configuration // 配置类
@ConditionalOnClass(UserService.class) // 有UserService才生效
@ConditionalOnProperty(prefix = "user.starter", name = "enabled", havingValue = "true", matchIfMissing = true) // 配置开关
public class UserAutoConfiguration {

    // 没有自定义UserService时，加载默认实现
    @Bean
    @ConditionalOnMissingBean
    public UserService userService() {
        return new UserService();
    }
}

// 步骤2：配置自动配置加载路径
// 创建文件：META-INF/spring/org.springframework.boot.autoconfigure.AutoConfiguration.imports
// 内容：com.boot.starter.user.config.UserAutoConfiguration

// 步骤3：打包为Maven依赖，其他项目引入即可自动配置
❗ 关键避坑：
1. 自定义自动配置类要放在启动类扫描范围外，避免重复加载；
2. @ConditionalOnMissingBean要放在@Bean注解上，确保用户自定义Bean优先；
3. SpringBoot3.x中自动配置文件路径已从spring.factories改为org.springframework.boot.autoconfigure.AutoConfiguration.imports。

三、外部化配置进阶（企业级配置管理）

解决：开发/测试/生产环境配置隔离、敏感配置加密、配置优先级问题。

3.1 多环境配置（企业标配）

核心：拆分配置文件，通过spring.profiles.active激活对应环境。

3.3 配置优先级（解决配置冲突）

SpringBoot加载配置的优先级（从高到低），企业级排错必懂：

  1. 命令行参数（--server.port=8081）
  2. 环境变量（如SPRING_DATASOURCE_PASSWORD）
  3. application-{profile}.yml（激活的环境配置）
  4. application.yml（公共配置）
  5. application.properties
  6. @PropertySource注解加载的配置
  7. 默认配置（SpringBoot内置）
✅ 实战技巧：生产环境优先用「环境变量+命令行参数」注入敏感配置，避免配置文件泄露。

四、SpringBoot 监控（Actuator）进阶

企业级项目必须有监控，Actuator是SpringBoot内置的监控工具，能实时查看项目状态。

4.1 基础配置（开启监控）

// 步骤1：导入依赖（pom.xml）

    org.springframework.boot
    spring-boot-starter-actuator


// 步骤2：配置监控端点（application.yml）
management:
  endpoints:
    web:
      exposure:
        include: "*" # 暴露所有端点（生产环境建议只暴露必要的）
      base-path: /actuator # 监控接口前缀（默认/actuator）
  endpoint:
    health:
      show-details: always # 显示健康详情（生产环境建议when_authorized）
    shutdown:
      enabled: true # 允许远程关闭（生产环境禁用！）
    metrics:
      enabled: true # 开启指标监控
  metrics:
    tags:
      application: boot-demo # 标记应用名称，多实例时区分

4.2 常用监控接口（企业级必查）

接口地址 功能 生产环境用途
/actuator/health 健康状态（数据库/Redis/磁盘等） 检查服务是否正常运行
/actuator/info 应用信息（版本、名称） 确认部署的版本是否正确
/actuator/beans IOC容器中所有Bean 排查Bean是否加载、是否重复
/actuator/metrics JVM/CPU/内存/请求指标 性能监控、瓶颈排查
/actuator/threaddump 线程快照 排查死锁、线程阻塞
/actuator/loggers 日志级别配置 生产环境动态调整日志级别（无需重启）

4.3 自定义监控指标（企业级进阶）

场景：监控业务指标（如接口调用次数、订单量），实现自定义告警。

// 步骤1：自定义指标收集器
package com.boot.demo.metrics;

import io.micrometer.core.instrument.Counter;
import io.micrometer.core.instrument.MeterRegistry;
import jakarta.annotation.PostConstruct;
import org.springframework.stereotype.Component;

@Component
public class CustomMetrics {
    private Counter userLoginCounter; // 用户登录次数计数器

    // 注入MeterRegistry（Actuator内置）
    private final MeterRegistry meterRegistry;

    public CustomMetrics(MeterRegistry meterRegistry) {
        this.meterRegistry = meterRegistry;
    }

    // 初始化自定义指标
    @PostConstruct
    public void init() {
        userLoginCounter = meterRegistry.counter("user.login.count", "module", "user");
    }

    // 计数方法（登录接口调用时调用）
    public void incrementLoginCount() {
        userLoginCounter.increment();
    }
}

// 步骤2：业务接口中使用
package com.boot.demo.controller;

import com.boot.demo.metrics.CustomMetrics;
import org.springframework.beans.factory.annotation.Autowired;
import org.springframework.web.bind.annotation.PostMapping;
import org.springframework.web.bind.annotation.RequestMapping;
import org.springframework.web.bind.annotation.RestController;

@RestController
@RequestMapping("/user")
public class UserController {

    @Autowired
    private CustomMetrics customMetrics;

    @PostMapping("/login")
    public String login() {
        // 业务逻辑...
        customMetrics.incrementLoginCount(); // 登录次数+1
        return "success";
    }
}

// 步骤3：Redis配置类（序列化优化）
package com.boot.demo.config;

import org.springframework.context.annotation.Bean;
import org.springframework.context.annotation.Configuration;
import org.springframework.data.redis.connection.RedisConnectionFactory;
import org.springframework.data.redis.core.RedisTemplate;
import org.springframework.data.redis.serializer.GenericJackson2JsonRedisSerializer;
import org.springframework.data.redis.serializer.StringRedisSerializer;

@Configuration
public class RedisConfig {

    @Bean
    public RedisTemplate redisTemplate(RedisConnectionFactory factory) {
        RedisTemplate template = new RedisTemplate<>();
        template.setConnectionFactory(factory);
        
        // key序列化（String）
        StringRedisSerializer stringSerializer = new StringRedisSerializer();
        template.setKeySerializer(stringSerializer);
        template.setHashKeySerializer(stringSerializer);
        
        // value序列化（JSON）
        GenericJackson2JsonRedisSerializer jsonSerializer = new GenericJackson2JsonRedisSerializer();
        template.setValueSerializer(jsonSerializer);
        template.setHashValueSerializer(jsonSerializer);
        
        template.afterPropertiesSet();
        return template;
    }
}

// 步骤4：业务层使用Redis缓存
package com.boot.demo.service.impl;

import com.boot.demo.mapper.UserMapper;
import com.boot.demo.pojo.User;
import com.boot.demo.service.UserService;
import com.boot.demo.util.Result;
import org.springframework.beans.factory.annotation.Autowired;
import org.springframework.data.redis.core.RedisTemplate;
import org.springframework.stereotype.Service;

import java.util.concurrent.TimeUnit;

@Service
public class UserServiceImpl implements UserService {

    @Autowired
    private UserMapper userMapper;
    
    @Autowired
    private RedisTemplate redisTemplate;

    @Override
    public Result findById(Integer id) {
        String key = "user:info:" + id;
        // 1. 先查缓存
        User user = (User) redisTemplate.opsForValue().get(key);
        if (user != null) {
            return Result.success(user);
        }
        // 2. 缓存没有查数据库
        user = userMapper.findById(id);
        if (user == null) {
            return Result.fail("用户不存在");
        }
        // 3. 存入缓存（设置过期时间，避免缓存雪崩）
        redisTemplate.opsForValue().set(key, user, 30, TimeUnit.MINUTES);
        return Result.success(user);
    }
}

七、SpringBoot 进阶核心总结

✅ 原理层进阶核心：
1. 自动配置：SPI加载+@Conditional条件装配，自定义Starter复用通用功能；
2. 配置管理：多环境隔离+敏感配置加密+配置优先级，生产环境安全可控；
3. 监控：Actuator内置接口+自定义指标，实时掌握服务状态。
✅ 实战层进阶核心：
1. 组件整合：Redis缓存（减轻数据库压力）、Swagger接口文档（提效协作）；
2. 部署优化：JAR包瘦身+JVM调优+多实例部署，适配生产环境；
3. 性能优化：缓存+连接池+Tomcat线程数调整，提升服务吞吐量。
✅ 企业级开发关键：
1. 提效：利用SpringBoot自动配置，减少重复配置；
2. 稳定：监控+事务+全局异常，保证服务可用性；
3. 安全：配置加密+生产环境关闭Swagger+权限控制；
4. 性能：缓存+JVM调优+多实例，支撑高并发。
