10.1 Spring 核心：IOC/DI（控制反转/依赖注入）

10.1.1 先搞懂：IOC/DI 到底是啥？（大白话）

没有Spring时的问题：
你要调用UserService，得自己 new UserService()；UserService里要调用UserDao，得自己 new UserDaoImpl()。
→ 所有对象都要你手动new，耦合度极高（改Dao实现类要改所有new的地方）。

有了Spring后：
✨ IOC（控制反转）：对象的创建权交给Spring容器，你不用new，Spring帮你造；
✨ DI（依赖注入）：Spring自动把Dao对象“塞”进Service里，不用你手动赋值。

💡 一句话记住：
IOC = 把“new对象”的活儿交给Spring；
DI = Spring自动给对象的属性赋值。

② DAO层：com.dao.UserDao + UserDaoImpl

// 接口
package com.dao;
import com.pojo.User;
public interface UserDao {
    User findById(Integer id);
}

// 实现类
package com.dao.impl;
import com.dao.UserDao;
import com.pojo.User;
import org.springframework.stereotype.Repository; // 标记为Spring组件

// @Repository：告诉Spring，这个类交给容器管理
@Repository
public class UserDaoImpl implements UserDao {
    @Override
    public User findById(Integer id) {
        // 模拟数据库查询
        return new User(1, "Spring用户", "123456");
    }
}

③ Service层：com.service.UserService

package com.service;

import com.dao.UserDao;
import com.pojo.User;
import org.springframework.beans.factory.annotation.Autowired;
import org.springframework.stereotype.Service;

// @Service：标记为Spring服务组件
@Service
public class UserService {
    // @Autowired：DI依赖注入 → Spring自动把UserDaoImpl对象塞进来
    @Autowired
    private UserDao userDao;

    public User findUserById(Integer id) {
        return userDao.findById(id);
    }
}

10.2 SpringMVC 核心原理（小白能懂）

SpringMVC = Spring的Web模块，专门替代Servlet，简化Web开发（不用写Servlet、不用配置web.xml）。

SpringMVC 核心流程（7步，大白话）

  1. 浏览器发请求 → Tomcat接收；
  2. Tomcat把请求交给 DispatcherServlet（前端控制器）（SpringMVC核心，不用你写）；
  3. DispatcherServlet根据请求地址找 Controller（控制器）；
  4. Controller处理业务逻辑（调用Service）；
  5. Controller返回结果（页面/数据）给DispatcherServlet；
  6. DispatcherServlet找 视图解析器 处理页面跳转；
  7. 最终响应返回给浏览器。
💡 核心角色记忆：
DispatcherServlet = 前台接待员（所有请求都找它）；
Controller = 具体办事员（处理业务）；
视图解析器 = 导航员（负责页面跳转）。

10.4 SpringMVC 核心操作（纯实战）

10.4.1 第一个Controller（HelloWorld）

创建：com.controller.HelloController

package com.controller;

import org.springframework.stereotype.Controller;
import org.springframework.web.bind.annotation.GetMapping;
import org.springframework.web.bind.annotation.ResponseBody;

// @Controller：标记为SpringMVC控制器
@Controller
public class HelloController {

    // 1. 页面跳转（返回字符串=页面名，视图解析器拼接前缀后缀）
    @GetMapping("/hello") // 访问路径：http://localhost:8080/hello
    public String hello() {
        // 跳转到 /WEB-INF/pages/hello.jsp
        return "hello";
    }

    // 2. 返回JSON数据（@ResponseBody）
    @GetMapping("/hello/json")
    @ResponseBody // 不跳转页面，直接返回数据
    public String helloJson() {
        return "{\"msg\":\"Hello SpringMVC！适配JDK24+Tomcat11\"}";
    }
}

10.4.4 核心操作：参数接收（实战常用）

package com.controller;

import com.pojo.User;
import org.springframework.stereotype.Controller;
import org.springframework.web.bind.annotation.PostMapping;
import org.springframework.web.bind.annotation.RequestParam;
import org.springframework.web.bind.annotation.ResponseBody;

@Controller
public class UserController {

    // 1. 单个参数接收
    @PostMapping("/user/login")
    @ResponseBody
    public String login(@RequestParam("username") String username,
                        @RequestParam("password") String password) {
        return "用户名：" + username + "，密码：" + password;
    }

    // 2. 对象接收（自动封装，参数名=实体属性名）
    @PostMapping("/user/add")
    @ResponseBody
    public String addUser(User user) {
        return "添加用户：" + user;
    }
}

10.5 拦截器（Interceptor）（SpringMVC版登录拦截）

SpringMVC拦截器 = 增强版Filter，专门拦截Controller请求，可在请求前/后执行逻辑。

Step 1：编写拦截器类

创建：com.interceptor.LoginInterceptor

package com.interceptor;

import jakarta.servlet.http.HttpServletRequest;
import jakarta.servlet.http.HttpServletResponse;
import org.springframework.web.servlet.HandlerInterceptor;

// 登录拦截器
public class LoginInterceptor implements HandlerInterceptor {

    // 请求处理前执行（核心）
    @Override
    public boolean preHandle(HttpServletRequest request,
                             HttpServletResponse response,
                             Object handler) throws Exception {
        // 1. 获取Session中的登录用户
        Object user = request.getSession().getAttribute("loginUser");

        // 2. 未登录 → 跳转到登录页
        if (user == null) {
            response.sendRedirect("/login.jsp");
            return false; // 拦截请求
        }

        // 3. 已登录 → 放行
        return true;
    }
}

Step 3：测试拦截器

  1. 创建 src/main/webapp/login.jsp（登录页）；
  2. 未登录时访问 http://localhost:8080/hello → 自动跳转到login.jsp；
  3. 模拟登录：在Controller中设置Session → 访问hello可正常跳转。

10.6 本章核心总结（背会直接开发）

Spring + SpringMVC 手动搭建固定流程
  1. 导依赖：spring-context（IOC） + spring-webmvc（MVC） + jakarta.servlet-api（Tomcat11）；
  2. 写配置：applicationContext.xml（Spring） + spring-mvc.xml（MVC） + web.xml（DispatcherServlet）；
  3. 写代码：Controller（@Controller） + Service（@Service） + Dao（@Repository）；
  4. 配Tomcat：选择Tomcat11，部署war包，简化访问路径；
  5. 测试：启动Tomcat，访问Controller路径。

核心注解必背

  • 🍪 Spring核心：@Component/@Service/@Repository（IOC）、@Autowired（DI）；
  • 🔐 SpringMVC核心：@Controller（控制器）、@GetMapping/@PostMapping（请求映射）、@ResponseBody（返回JSON）；
  • ⚡ 拦截器：实现HandlerInterceptor，重写preHandle，配置拦截/放行路径。
✅ 适配你的环境关键：
1. Spring版本选6.1+（兼容JDK24+Tomcat11）；
2. web.xml版本为6.0（Jakarta EE 10）；
3. 所有依赖为jakarta开头，而非javax。
