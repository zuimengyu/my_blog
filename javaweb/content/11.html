11.1 SpringBoot 是什么？解决什么问题

大白话（实战版）：
SpringBoot = Spring+SpringMVC的简化升级版，用来解决之前Spring/ SpringMVC的繁琐配置问题。

之前Spring+SpringMVC的痛点（SpringBoot全解决）

  • 要手动配N多XML：web.xml、applicationContext.xml、spring-mvc.xml
  • 要手动导一堆依赖、处理版本冲突
  • 要手动配置Tomcat、部署项目
  • 配置繁琐、上手慢、易出错

SpringBoot 4大核心优势（企业必用）

  • ✅ 起步依赖：一键导入整套依赖，不用管版本
  • ✅ 自动配置：零XML，SpringBoot自动配Spring/MVC/Tomcat
  • ✅ 内嵌Tomcat：不用外置Tomcat，直接运行main方法启动项目
  • ✅ 准生产环境：内置监控、日志、异常处理
💡 一句话记住：
SpringBoot = 约定大于配置 + 一键整合 + 内嵌服务器
以前1小时配环境，现在1分钟搭建项目

Step 3：编写SpringBoot启动类（项目入口·唯一核心类）

创建包：com.example，新建启动类 SpringbootDemoApplication.java

package com.example;

import org.mybatis.spring.annotation.MapperScan;
import org.springframework.boot.SpringApplication;
import org.springframework.boot.autoconfigure.SpringBootApplication;

// @SpringBootApplication：SpringBoot核心注解，自动配置
// @MapperScan：扫描MyBatis的Mapper接口（衔接第9章）
@SpringBootApplication
@MapperScan("com.example.mapper")
public class SpringbootDemoApplication {
    // SpringBoot项目唯一入口：运行main方法即可启动项目（内嵌Tomcat11）
    public static void main(String[] args) {
        SpringApplication.run(SpringbootDemoApplication.class, args);
        System.out.println("✅ SpringBoot 启动成功！内嵌Tomcat11+JDK24");
    }
}
💡 革命性变化：
以前启动项目要配Tomcat、部署war包；
现在直接运行main方法，自动启动内嵌Tomcat11！

11.3 SpringBoot 核心配置（application.yml）

SpringBoot抛弃XML，用application.yml/application.properties配置，简洁易读。

Step 1：创建配置文件
  1. 目录：src/main/resources；
  2. 新建文件：application.yml（SpringBoot默认识别）；
Step 2：编写核心配置（端口+数据库+MyBatis）
# 服务器配置（内嵌Tomcat11）
server:
  port: 8080          # 访问端口
  servlet:
    context-path: /   # 项目访问路径

# 数据库配置（复用你的javaweb_test库）
spring:
  datasource:
    driver-class-name: com.mysql.cj.jdbc.Driver
    url: jdbc:mysql://localhost:3306/javaweb_test?serverTimezone=UTC&useSSL=false&characterEncoding=utf8
    username: root
    password: 123456   # 改成你的MySQL密码

# MyBatis配置
mybatis:
  mapper-locations: classpath:mapper/*.xml  # SQL映射文件路径
  type-aliases-package: com.example.pojo     # 实体类别名
  configuration:
    map-underscore-to-camel-case: true      # 下划线转驼峰

11.4 SpringBoot 整合 MyBatis（复用之前代码）

完全复用第9章的User、Mapper，零改动，直接整合！

1）实体类：com.example.pojo.User

package com.example.pojo;

import lombok.Data;

@Data
public class User {
    private Integer id;
    private String username;
    private String password;
    private Integer age;
    private String address;
}

2）Mapper接口：com.example.mapper.UserMapper

package com.example.mapper;

import com.example.pojo.User;
import org.apache.ibatis.annotations.Select;
import java.util.List;

public interface UserMapper {
    // 查询所有用户（注解版SQL，也可用XML）
    @Select("select * from user")
    List findAll();

    // 根据ID查询用户
    @Select("select * from user where id=#{id}")
    User findById(Integer id);
}
💡 SpringBoot+MyBatis支持两种写法：
1. 注解版（简单SQL，无XML）；2. XML版（复杂SQL，复用第9章）

3）Service层：com.example.service.UserService

package com.example.service;

import com.example.mapper.UserMapper;
import com.example.pojo.User;
import org.springframework.beans.factory.annotation.Autowired;
import org.springframework.stereotype.Service;
import java.util.List;

@Service
public class UserService {
    @Autowired
    private UserMapper userMapper;

    // 查询所有用户
    public List findAll() {
        return userMapper.findAll();
    }

    // 根据ID查询用户
    public User findById(Integer id) {
        return userMapper.findById(id);
    }
}

11.5 SpringBoot 实现 RESTful 接口（前后端分离标配）

RESTful：用HTTP动词表示操作（GET查询、POST添加、PUT修改、DELETE删除），返回JSON数据

1）Controller层：com.example.controller.UserController

package com.example.controller;

import com.example.pojo.User;
import com.example.service.UserService;
import org.springframework.beans.factory.annotation.Autowired;
import org.springframework.web.bind.annotation.GetMapping;
import org.springframework.web.bind.annotation.PathVariable;
import org.springframework.web.bind.annotation.RequestMapping;
import org.springframework.web.bind.annotation.RestController;

import java.util.List;

// @RestController = @Controller + @ResponseBody（直接返回JSON，不跳转页面）
@RestController
@RequestMapping("/user") // 统一前缀
public class UserController {
    @Autowired
    private UserService userService;

    // 1. GET查询：查询所有用户
    @GetMapping("/list")
    public List findAll() {
        return userService.findAll();
    }

    // 2. GET查询：根据ID查询用户
    @GetMapping("/{id}")
    public User findById(@PathVariable Integer id) {
        return userService.findById(id);
    }
}

2）启动测试（零配置·直接运行）

  1. 找到启动类 SpringbootDemoApplication；
  2. 右键 Run 'SpringbootDemoApplication'；
  3. 控制台看到 Tomcat started on port 8080 即启动成功；
  4. 浏览器访问：
    • 所有用户：http://localhost:8080/user/list
    • ID=1用户：http://localhost:8080/user/1
  5. 直接返回JSON数据，前后端分离标配！
❗ 小白必坑：
1. 启动类要在最外层包（com.example），否则扫描不到Controller/Service；
2. MySQL服务必须启动，否则连接数据库失败；
3. 密码改成你自己的MySQL密码。

11.6 本章核心总结（背会直接写企业项目）

SpringBoot 纯手动开发固定5步（社区版通用）
  1. 创建纯Maven项目，导入SpringBoot父工程+web起步依赖；
  2. 编写启动类，加@SpringBootApplication，运行main方法；
  3. 写application.yml配置（端口、数据库、MyBatis）；
  4. 写代码：Mapper（@Mapper）→ Service（@Service）→ Controller（@RestController）；
  5. 启动项目，访问RESTful接口。

核心注解必背（SpringBoot3.x）

  • 🍪 @SpringBootApplication：启动类核心，自动配置
  • 🔐 @RestController：RESTful接口，返回JSON
  • ⚡ @GetMapping/@PostMapping：请求映射
  • 🎯 @Service/@Autowired：业务层+依赖注入
  • 📊 @MapperScan：扫描MyBatis接口
