第9章：MyBatis 快速上手（纯实战版）

9.1 MyBatis 是什么？解决什么问题

大白话（实战版）：
MyBatis 是一款半自动ORM持久层框架，专门用来简化JDBC代码，不用再写连接、预编译、结果集封装、关闭资源等重复代码。

原生JDBC的痛点（MyBatis全解决）

  • 重复写：注册驱动、获取连接、关闭资源
  • 手动封装 ResultSet → Java对象，繁琐易错
  • SQL硬编码在Java里，修改麻烦
  • 参数拼接易出现SQL注入

MyBatis 核心优势（企业必用）

  • SQL写在XML里，解耦、易维护
  • 自动封装结果集，一行代码转Java对象
  • 安全参数绑定，防SQL注入
  • 轻量、简单、上手快
✅ 一句话定位：MyBatis = 简化版JDBC + SQL分离 + 自动映射

package com.pojo;

public class User {
    private Integer id;
    private String username;
    private String password;
    private Integer age;
    private String address;

    // 无参/有参构造、get/set、toString
    public User() {}
    public User(Integer id, String username, String password, Integer age, String address) {
        this.id = id;
        this.username = username;
        this.password = password;
        this.age = age;
        this.address = address;
    }

    // 自行生成getter/setter/toString
    public Integer getId() {return id;}
    public void setId(Integer id) {this.id = id;}
    public String getUsername() {return username;}
    public void setUsername(String username) {this.username = username;}
    public String getPassword() {return password;}
    public void setPassword(String password) {this.password = password;}
    public Integer getAge() {return age;}
    public void setAge(Integer age) {this.age = age;}
    public String getAddress() {return address;}
    public void setAddress(String address) {this.address = address;}

    @Override
    public String toString() {
        return "User{id=" + id + ", username='" + username + "'}";
    }
}

4）Mapper接口（com.mapper.UserMapper，等价于Dao）

package com.mapper;

import com.pojo.User;
import org.apache.ibatis.annotations.Param;
import java.util.List;

// MyBatis Mapper接口（只需定义方法，无需实现类）
public interface UserMapper {
    // 1. 根据id查询用户
    User findById(Integer id);

    // 2. 查询所有用户
    List findAll();

    // 3. 添加用户
    int add(User user);

    // 4. 修改用户
    int update(User user);

    // 5. 根据id删除用户
    int deleteById(Integer id);

    // 6. 多条件查询（实战常用）
    List findByCondition(@Param("username") String username, @Param("age") Integer age);
}

6）MyBatis 工具类（com.util.MyBatisUtil，企业标配）

package com.util;

import org.apache.ibatis.io.Resources;
import org.apache.ibatis.session.SqlSession;
import org.apache.ibatis.session.SqlSessionFactory;
import org.apache.ibatis.session.SqlSessionFactoryBuilder;
import java.io.IOException;
import java.io.InputStream;

// MyBatis 工具类：获取SqlSession（等价于数据库连接）
public class MyBatisUtil {
    private static SqlSessionFactory sqlSessionFactory;

    // 静态代码块：项目启动时加载一次
    static {
        try {
            String resource = "mybatis-config.xml";
            InputStream inputStream = Resources.getResourceAsStream(resource);
            sqlSessionFactory = new SqlSessionFactoryBuilder().build(inputStream);
        } catch (IOException e) {
            e.printStackTrace();
        }
    }

    // 获取SqlSession（autoCommit=true：自动提交事务）
    public static SqlSession getSqlSession() {
        return sqlSessionFactory.openSession(true);
    }

    // 关闭SqlSession
    public static void close(SqlSession session) {
        if (session != null) session.close();
    }
}

9.3 MyBatis 核心操作（纯实战CRUD）

❗ 实战必坑：
1. #{}：预编译，安全，防SQL注入（99%场景用这个）
2. ${}：字符串拼接，不安全，仅用于表名/排序（极少用）

1）查询单条（根据ID）

@Test
public void testFindById() {
    SqlSession session = MyBatisUtil.getSqlSession();
    UserMapper mapper = session.getMapper(UserMapper.class);
    User user = mapper.findById(1);
    System.out.println(user);
    MyBatisUtil.close(session);
}

参数传递总结（实战必背）

  • 单个参数：直接用 #{参数名}
  • 对象参数：用 #{属性名}
  • 多个参数：接口加 @Param("别名")，XML用 #{别名}

9.4 MyBatis 动态SQL（只学3个实战标签）

动态SQL：根据条件自动拼接SQL，解决多条件查询痛点

1）<if>：条件判断（最常用）

2）<where>：自动处理where/and/or，避免语法错误

3）<foreach>：遍历集合（批量查询/删除）

实战：多条件动态查询（用户搜索）

① Mapper接口追加方法

// 多条件查询：用户名模糊查询 + 年龄大于
List findByCondition(@Param("username") String username, @Param("age") Integer age);

② UserMapper.xml 追加动态SQL

③ 测试动态查询

@Test
public void testDynamic() {
    SqlSession session = MyBatisUtil.getSqlSession();
    UserMapper mapper = session.getMapper(UserMapper.class);
    // 只传用户名：模糊查询
    List list = mapper.findByCondition("a", null);
    // 传用户名+年龄：两个条件
    // List list = mapper.findByCondition("a", 20);
    list.forEach(System.out::println);
    MyBatisUtil.close(session);
}
✅ 动态SQL优势：
传什么参数就拼什么SQL，不用写N个if-else，代码极简、易维护

已删除10%极少用动态标签（完全不用学）

trim、set、choose、when、otherwise（企业99%不用）

9.5 本章实战总结（背会直接写项目）

企业MyBatis开发固定5步
  1. 导依赖：MyBatis + MySQL驱动
  2. 写配置：mybatis-config.xml（数据库+映射文件）
  3. 写接口：Mapper接口（方法名对应SQL id）
  4. 写SQL：Mapper.xml（namespace绑定接口）
  5. 调用：工具类获取SqlSession → getMapper → 调用方法

核心必背

  • 🍪 #{} 安全预编译，实战永远用这个
  • 🔐 动态SQL：where + if 搞定90%多条件查询
  • ⚡ 无实现类，MyBatis自动生成代理对象
  • 🎯 解耦：SQL写在XML，Java代码只负责调用
