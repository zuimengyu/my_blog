第7章：JDBC 进阶 & 数据库连接池（小白完整版）

7.0 前置核心：原生JDBC有啥问题？

大白话： 之前写的原生JDBC，每次请求都要「创建连接→执行SQL→关闭连接」，创建/关闭连接特别耗性能，就像每次喝水都要「造杯子→喝水→扔杯子」，太浪费！

❗ 原生JDBC的3大痛点：
1. 频繁创建/关闭连接 → 性能极低
2. 硬编码（连接参数写死在代码里）→ 改参数要改代码、重新打包
3. 无连接限制 → 并发高时数据库崩掉
💡 解决方案：
📌 数据库连接池 = 提前造好一批杯子（连接），放在池子里，用的时候拿，用完放回，不用重复造！

7.1 数据库连接池核心（小白必懂）

术语：数据库连接池（Connection Pool）
👉 大白话：存放数据库连接的「池子」，核心规则：

  1. 项目启动 → 连接池创建N个连接（预设数量）
  2. 程序需要连接 → 从池子里拿（不用新建）
  3. 程序用完连接 → 放回池子（不关闭）
  4. 项目关闭 → 连接池销毁所有连接

常用连接池（90%项目用这2个）：
1. Druid（阿里开源，性能最好、功能最全，企业首选）
2. HikariCP（SpringBoot默认，轻量快速）

💡 小白先学 Druid，企业用得最多！

7.2.2 连接池配置文件（db.properties）

在 src/main/resources 下新建 db.properties（解耦硬编码）：

# MySQL 连接参数
jdbc.driverClass=com.mysql.cj.jdbc.Driver
jdbc.url=jdbc:mysql://localhost:3306/javaweb_test?serverTimezone=UTC&useSSL=false&characterEncoding=utf8
jdbc.username=root
jdbc.password=123456

# Druid 连接池参数（小白不用改，默认即可）
druid.initialSize=5        # 初始连接数
druid.maxActive=20         # 最大连接数
druid.minIdle=5            # 最小空闲连接数
druid.maxWait=3000         # 获取连接的最大等待时间（毫秒）

7.2.3 Druid 工具类（DruidUtil.java）

package com.util;

import com.alibaba.druid.pool.DruidDataSourceFactory;
import javax.sql.DataSource;
import java.sql.Connection;
import java.sql.ResultSet;
import java.sql.PreparedStatement;
import java.util.Properties;

// Druid 连接池工具类（企业标配）
public class DruidUtil {
    // 静态数据源（全局唯一）
    private static DataSource dataSource;

    // 静态代码块：项目启动时加载配置、初始化连接池
    static {
        try {
            // 1. 加载配置文件
            Properties props = new Properties();
            props.load(DruidUtil.class.getClassLoader().getResourceAsStream("db.properties"));

            // 2. 创建Druid连接池
            dataSource = DruidDataSourceFactory.createDataSource(props);
            System.out.println("Druid连接池初始化成功！");
        } catch (Exception e) {
            throw new RuntimeException("连接池初始化失败！", e);
        }
    }

    // 1. 获取连接（从池子里拿，不用新建）
    public static Connection getConn() throws Exception {
        return dataSource.getConnection();
    }

    // 2. 获取数据源（供后续框架使用）
    public static DataSource getDataSource() {
        return dataSource;
    }

    // 3. 关闭资源（归还连接到池子，不是真关闭）
    public static void close(Connection conn, PreparedStatement pstmt, ResultSet rs) {
        try {
            if (rs != null) rs.close();
            if (pstmt != null) pstmt.close();
            if (conn != null) conn.close(); // 归还连接到池子里
        } catch (Exception e) {
            e.printStackTrace();
        }
    }

    // 重载：增删改关闭2个资源
    public static void close(Connection conn, PreparedStatement pstmt) {
        close(conn, pstmt, null);
    }
}
关键区别：conn.close() 不再是关闭连接，而是把连接归还到连接池！

7.2.4 测试 Druid 连接池

package com.test;

import com.util.DruidUtil;
import java.sql.Connection;

// 测试Druid连接池
public class DruidTest {
    public static void main(String[] args) throws Exception {
        // 多次获取连接，看是否复用
        for (int i = 0; i < 10; i++) {
            Connection conn = DruidUtil.getConn();
            System.out.println("第" + (i+1) + "次获取连接：" + conn);
            DruidUtil.close(conn, null); // 归还连接
        }
    }
}
💡 效果：打印的连接对象地址大部分重复 → 说明复用了连接，性能提升！

7.3 DAO 模式封装（解决代码重复）

术语：DAO（Data Access Object）数据访问对象
👉 大白话：把数据库操作（增删改查）封装成专门的类，Service层不用写JDBC代码，只调用DAO方法。

7.3.1 先建实体类（User.java）

package com.pojo;

// 用户实体类（对应数据库user表）
public class User {
    private Integer id;
    private String username;
    private String password;
    private Integer age;
    private String address;

    // 无参构造（必须有，后续框架要用）
    public User() {}

    // 有参构造、get/set方法、toString方法（小白直接生成）
    public User(Integer id, String username, String password, Integer age, String address) {
        this.id = id;
        this.username = username;
        this.password = password;
        this.age = age;
        this.address = address;
    }

    public Integer getId() { return id; }
    public void setId(Integer id) { this.id = id; }
    public String getUsername() { return username; }
    public void setUsername(String username) { this.username = username; }
    public String getPassword() { return password; }
    public void setPassword(String password) { this.password = password; }
    public Integer getAge() { return age; }
    public void setAge(Integer age) { this.age = age; }
    public String getAddress() { return address; }
    public void setAddress(String address) { this.address = address; }

    @Override
    public String toString() {
        return "User{" +
                "id=" + id +
                ", username='" + username + '\'' +
                ", password='" + password + '\'' +
                ", age=" + age +
                ", address='" + address + '\'' +
                '}';
    }
}

7.3.2 DAO 接口（UserDao.java）

package com.dao;

import com.pojo.User;
import java.util.List;

// UserDAO接口（定义操作规范）
public interface UserDao {
    // 根据用户名查询用户
    User findByUsername(String username);
    // 查询所有用户
    List<User> findAll();
    // 添加用户
    int add(User user);
    // 修改用户
    int update(User user);
    // 删除用户
    int delete(Integer id);
}

7.3.3 DAO 实现类（UserDaoImpl.java）

package com.dao.impl;

import com.dao.UserDao;
import com.pojo.User;
import com.util.DruidUtil;
import java.sql.Connection;
import java.sql.PreparedStatement;
import java.sql.ResultSet;
import java.util.ArrayList;
import java.util.List;

// UserDAO实现类（真正的数据库操作）
public class UserDaoImpl implements UserDao {
    // 1. 根据用户名查询用户
    @Override
    public User findByUsername(String username) {
        Connection conn = null;
        PreparedStatement pstmt = null;
        ResultSet rs = null;
        User user = null;

        try {
            conn = DruidUtil.getConn();
            String sql = "SELECT * FROM user WHERE username=?";
            pstmt = conn.prepareStatement(sql);
            pstmt.setString(1, username);
            rs = pstmt.executeQuery();

            // 封装结果到User对象
            if (rs.next()) {
                user = new User();
                user.setId(rs.getInt("id"));
                user.setUsername(rs.getString("username"));
                user.setPassword(rs.getString("password"));
                user.setAge(rs.getInt("age"));
                user.setAddress(rs.getString("address"));
            }
        } catch (Exception e) {
            e.printStackTrace();
        } finally {
            DruidUtil.close(conn, pstmt, rs);
        }
        return user;
    }

    // 2. 查询所有用户（其余方法小白可自行实现，逻辑类似）
    @Override
    public List<User> findAll() {
        Connection conn = null;
        PreparedStatement pstmt = null;
        ResultSet rs = null;
        List<User> userList = new ArrayList<>();

        try {
            conn = DruidUtil.getConn();
            String sql = "SELECT * FROM user";
            pstmt = conn.prepareStatement(sql);
            rs = pstmt.executeQuery();

            while (rs.next()) {
                User user = new User();
                user.setId(rs.getInt("id"));
                user.setUsername(rs.getString("username"));
                user.setPassword(rs.getString("password"));
                user.setAge(rs.getInt("age"));
                user.setAddress(rs.getString("address"));
                userList.add(user);
            }
        } catch (Exception e) {
            e.printStackTrace();
        } finally {
            DruidUtil.close(conn, pstmt, rs);
        }
        return userList;
    }

    // 3. 添加用户
    @Override
    public int add(User user) {
        Connection conn = null;
        PreparedStatement pstmt = null;
        int row = 0;

        try {
            conn = DruidUtil.getConn();
            String sql = "INSERT INTO user(username,password,age,address) VALUES(?,?,?,?)";
            pstmt = conn.prepareStatement(sql);
            pstmt.setString(1, user.getUsername());
            pstmt.setString(2, user.getPassword());
            pstmt.setInt(3, user.getAge());
            pstmt.setString(4, user.getAddress());
            row = pstmt.executeUpdate();
        } catch (Exception e) {
            e.printStackTrace();
        } finally {
            DruidUtil.close(conn, pstmt);
        }
        return row;
    }

    // 4. 修改用户（略，逻辑和添加类似）
    @Override
    public int update(User user) { return 0; }

    // 5. 删除用户（略，逻辑和添加类似）
    @Override
    public int delete(Integer id) { return 0; }
}

7.3.4 Service 层调用 DAO（UserService.java）

package com.service;

import com.dao.UserDao;
import com.dao.impl.UserDaoImpl;
import com.pojo.User;
import java.util.List;

// 业务层：调用DAO，处理业务逻辑（如登录验证）
public class UserService {
    // 注入DAO（后续用Spring自动注入，现在手动new）
    private UserDao userDao = new UserDaoImpl();

    // 登录验证业务
    public boolean login(String username, String password) {
        // 调用DAO查询用户
        User user = userDao.findByUsername(username);
        // 验证密码
        return user != null && user.getPassword().equals(password);
    }

    // 查询所有用户
    public List<User> findAll() {
        return userDao.findAll();
    }
}

7.4 本章终极总结（小白记住这些就够）

  • 🍉 连接池核心：Druid 提前创建连接，复用连接，提升性能，解决原生JDBC痛点
  • 🔐 DAO模式：接口+实现类，把数据库操作封装成专门的类，解耦业务和数据访问
  • ⚡ 企业套路：Druid连接池 + DAO封装 + 实体类映射数据库表
  • 🎯 配置文件：db.properties 解耦硬编码，改参数不用改代码
