第5章：JSP 核心（小白完整版·Maven+Tomcat11）

5.0 前置核心：JSP 是干嘛的？

术语：JSP（Java Server Pages）Java 服务器页面
👉 大白话：能直接写 Java 代码的 HTML 页面。

为啥要用 JSP？
之前我们在 Servlet 里输出 HTML，要写一堆 out.write("<h1>...</h1>")，又臭又长、难维护。
JSP 直接在 HTML 里嵌 Java 代码，写页面和逻辑一步到位。

💡 最通俗类比：
📌 HTML = 纯静态页面（只能看，不能跑Java）
📌 Servlet = 纯Java代码（能跑逻辑，写页面麻烦）
📌 JSP = HTML + Java 合体（既能写页面，又能跑逻辑）

核心真相： JSP 本质就是一个 Servlet，Tomcat 会把 JSP 编译成 Java 类运行。

5.1 Maven 依赖配置（必须加，否则JSP报错）

Tomcat11 对应 JSP3.1 规范，必须用 jakarta 包，在 pom.xml 加依赖：

JSP 核心 Maven 坐标（Tomcat11专用） 
<!-- JSP 3.1 API（必须加） -->
<dependency>
    <groupId>jakarta.servlet.jsp</groupId>
    <artifactId>jakarta.servlet.jsp-api</artifactId>
    <version>3.1.0</version>
    <scope>provided</scope>
</dependency>

<!-- JSTL 标签库（后面常用，一起加上） -->
<dependency>
    <groupId>jakarta.servlet.jsp.jstl</groupId>
    <artifactId>jakarta.servlet.jsp.jstl-api</artifactId>
    <version>3.0.0</version>
</dependency>
<dependency>
    <groupId>org.glassfish.web</groupId>
    <artifactId>jakarta.servlet.jsp.jstl</artifactId>
    <version>3.0.1</version>
</dependency>
❗ 小白必坑：
1. 必须是 jakarta.servlet.jsp 包（Tomcat11强制）
2. <scope>provided</scope> 不能删（Tomcat自带JSP）

5.2 JSP 基础结构 & 第一个页面

5.2.1 JSP 页面存放位置

Maven 项目：src/main/webapp/ 目录下（和 HTML 放一起）

5.2.2 JSP 标准结构

<%@ page contentType="text/html;charset=UTF-8" language="java" %>
<!DOCTYPE html>
<html>
<head>
    <meta charset="UTF-8">
    <title>第一个JSP</title>
</head>
<body>
    <!-- HTML 内容 -->
    <h1>Hello JSP</h1>

    <!-- Java 代码（嵌在HTML里） -->
    <%
        String msg = "我是JSP里的Java代码";
        out.print(msg);
    %>
</body>
</html>

5.2.3 第一个 JSP 实战（index.jsp）

直接复制到 webapp/index.jsp

<%@ page contentType="text/html;charset=UTF-8" language="java" %>
<html>
<head>
    <title>第一个JSP</title>
</head>
<body>
    <h1>这是 JSP 页面</h1>
    <%
        // Java 代码：定义变量
        int a = 10;
        int b = 20;
        int sum = a + b;
    %>
    <p>10 + 20 = <%= sum %></p> <!-- 输出变量 -->
</body>
</html>
💡 访问方式：和HTML一样
http://localhost:8080/项目名/index.jsp

5.3 JSP 3种脚本元素（嵌Java代码的方式）

JSP 里写 Java 代码，只有 3 种写法，全部是常用：

<
写法 格式 作用（大白话） 常用度
脚本段 <% Java代码 %> 写任意Java逻辑（变量、判断、循环） ⭐⭐⭐⭐⭐
表达式 <%= 变量/值 %> 直接输出内容到页面 ⭐⭐⭐⭐⭐
声明 <%! 方法/变量 %> 定义全局变量、方法 ⭐⭐⭐

5.3.1 脚本段 <% %>（最常用）

<%
    // 写任意Java代码
    String name = "Java小白";
    int age = 20;
    if(age >= 18){
        out.print("成年");
    }
%>

5.3.2 表达式 <%= %>（最常用）

<%
    String name = "Java小白";
%>
<p>姓名：<%= name %></p> <!-- 等价于 out.print(name) -->

5.3.3 声明 <%! %>（常用）

<%!
    // 定义全局变量
    public static final String SITE = "JavaWeb学习";
    // 定义方法
    public int add(int x,int y){
        return x + y;
    }
%>

<!-- 使用 -->
<p>网站：<%= SITE %></p>
<p>3+5=<%= add(3,5) %></p>

5.4 JSP 内置对象（不用new，直接用）

术语：内置对象 → 大白话：Tomcat 提前创建好的对象，直接用，不用new。

共9个，只学6个常用的，3个极少用（标注）：

对象名 作用（大白话） 常用度
request 请求对象（取参数、存数据） ⭐⭐⭐⭐⭐
response 响应对象（跳转、输出） ⭐⭐⭐⭐
session 会话对象（存登录状态） ⭐⭐⭐⭐⭐
application 全局对象（整个项目共享数据） ⭐⭐⭐
out 输出对象（打印到页面） ⭐⭐⭐⭐
pageContext 页面上下文（存当前页面数据） ⭐⭐⭐
page 当前JSP对象（极少用） ⭐
config 配置对象（极少用） ⭐
exception 异常对象（极少用） ⭐

5.4.1 常用内置对象实战（直接复制）

<%@ page contentType="text/html;charset=UTF-8" %>
<html>
<body>
    <%
        // 1. request 存数据
        request.setAttribute("reqMsg", "我是request数据");
        // 2. session 存数据（登录必用）
        session.setAttribute("loginUser", "admin");
        // 3. application 存数据
        application.setAttribute("appMsg", "我是全局数据");
    %>

    <p>request数据：<%= request.getAttribute("reqMsg") %></p>
    <p>登录用户：<%= session.getAttribute("loginUser") %></p>
    <p>全局数据：<%= application.getAttribute("appMsg") %></p>
</body>
</html>

5.5 JSP 四大域对象（存数据的4个范围）

术语：域对象 → 大白话：存数据的盒子，范围不同。

域对象 范围（大白话） 使用场景
pageContext 当前页面 页面内临时数据
request 一次请求（转发共享） 表单数据、查询结果
session 一次会话（浏览器打开到关闭） 登录状态、用户信息
application 整个项目运行期间 全局统计、公共数据
💡 小白记住：
存数据：setAttribute("key", 值)
取数据：getAttribute("key")

5.6 EL 表达式（简化取数据，必学）

术语：EL（Expression Language）表达式
👉 大白话：不用写Java代码，就能取域对象里的数据。

5.6.1 EL 基本语法

${key}

5.6.2 EL 对比传统写法（简化10倍）

<%-- 传统JSP写法（麻烦） --%>
<%= session.getAttribute("loginUser") == null ? "未登录" : session.getAttribute("loginUser") %>

<%-- EL写法（简单） --%>
${loginUser}

5.6.3 EL 实战（取域数据）

<%
    request.setAttribute("username", "EL表达式");
    session.setAttribute("age", 20);
%>

<p>用户名：${username}</p>
<p>年龄：${age}</p>
<p>判断是否成年：${age >= 18}</p>
💡 EL 会自动从四大域里找数据，不用指定域，自动匹配。

5.7 JSTL 核心标签（替代Java判断、循环）

术语：JSTL（JSP标准标签库）
👉 大白话：用标签替代JSP里的Java代码，页面更干净。

5.7.1 引入JSTL标签（页面顶部加）

<%@ taglib prefix="c" uri="http://java.sun.com/jsp/jstl/core" %>

5.7.2 JSTL 常用标签（只学2个最常用）

1. 循环标签 <c:forEach>（遍历列表，必用）

<%
    // Servlet里存的列表，JSP直接遍历
    request.setAttribute("userList", new String[]{"张三","李四","王五"});
%>

<%-- 循环遍历 --%>
<c:forEach items="${userList}" var="user">
    <p>用户：${user}</p>
</c:forEach>

2. 判断标签 <c:if>（条件判断，必用）

<c:if test="${age >= 18}">
    <p>已成年</p>
</c:if>

5.8 综合实战：登录状态展示（最常用）

需求：

  • 从Session取登录用户
  • EL显示用户名
  • JSTL判断是否登录

完整代码（home.jsp）

<%@ page contentType="text/html;charset=UTF-8" %>
<%@ taglib prefix="c" uri="http://java.sun.com/jsp/jstl/core" %>
<html>
<head>
    <title>个人主页</title>
</head>
<body>
    <h1>个人中心</h1>

    <%-- JSTL+EL 判断登录 --%>
    <c:if test="${empty loginUser}">
        <p>您还未登录，请<a href="/login.jsp">登录</a></p>
    </c:if>

    <c:if test="${not empty loginUser}">
        <p>欢迎您：${loginUser}</p>
        <p><a href="/logout">退出登录</a></p>
    </c:if>
</body>
</html>

5.9 本章终极总结（小白记住这些就够）

🍉 JSP 基础

  • JSP = HTML + Java，本质是 Servlet
  • 3种脚本：<%代码%>、<%=输出%>、<%!声明%>

🔐 内置对象（常用6个）

  • request、response、session、application、out、pageContext
  • session：存登录状态（最核心）

⚡ EL 表达式（简化取数据）

  • 语法：${key}
  • 自动从四大域取数据，不用写Java

📝 JSTL 标签（替代Java逻辑）

  • <c:forEach>：循环
  • <c:if>：判断

🎯 实战套路

  1. Servlet 存数据到域（request/session）
  2. JSP 用 EL+JSTL 展示数据
  3. 页面干净，无Java代码
✅ 本章只删除以下极少用内容：
1. JSP 异常页面指令（page指令的errorPage）
2. 3个极少用内置对象（page、config、exception）
3. JSP 静态包含、动态包含（进阶用法）
4. JSTL 其他冷门标签（格式化、XML等）
