第1章：JavaWeb 前置认知（适配IDEA2025/JDK24/Tomcat11）

1.1 什么是 JavaWeb？

JavaWeb 是基于 Java 语言、遵循 HTTP 协议规范，用于开发B/S架构（浏览器/服务器）应用的技术体系总称，核心是通过 Java 代码处理客户端（浏览器/APP/小程序）的 HTTP 请求，并返回响应数据。

1.1.1 JavaWeb 核心技术栈

  • 基础层：JDK（你使用的JDK24）、Servlet API（Tomcat11内置Servlet 6.0规范）、JSP
  • 框架层：Spring、SpringMVC、MyBatis、SpringBoot
  • 容器层：Tomcat（你使用的11.0.11）、Jetty、JBoss
  • 数据层：MySQL（你使用的8.0+）、JDBC、连接池（Druid/HikariCP）

1.1.2 JavaWeb 典型应用场景

  • 企业后台管理系统（如ERP、OA、CRM）
  • 电商平台（商品展示、下单、支付接口）
  • 移动端APP/小程序的后端接口（RESTful API）
  • 数据可视化平台、日志分析系统

1.2 Web 核心概念（必掌握）

1.2.1 客户端 & 服务器

核心定义
  • 客户端（Client）：发起请求的终端，如Chrome浏览器、手机APP、Postman（接口测试工具），你的环境中主要是Windows11下的浏览器
  • 服务器（Server）：接收并处理请求、返回响应的服务端程序/硬件，你使用的Tomcat11是「Web应用服务器」，负责运行JavaWeb代码
  • 通信流程：客户端 → 发送HTTP请求 → 服务器（Tomcat） → 调用Java代码处理 → 服务器返回HTTP响应 → 客户端

1.2.2 HTTP 协议（超文本传输协议）

HTTP是客户端与服务器之间的应用层通信协议，特点：无状态（每次请求独立，服务器不记录客户端状态）、基于TCP/IP、明文传输。

① HTTP 请求结构
请求行：GET /index.html HTTP/1.1  （请求方法 + 请求路径 + 协议版本）
请求头：Host: localhost:8080      （键值对，如User-Agent、Content-Type、Cookie等）
        User-Agent: Chrome/124.0.0.0
        Content-Type: application/json
空行：（分隔请求头和请求体）
请求体：username=admin&password=123 （POST请求的参数，GET请求无请求体）
② HTTP 响应结构
状态行：HTTP/1.1 200 OK  （协议版本 + 状态码 + 状态描述）
响应头：Content-Type: text/html;charset=UTF-8 （响应内容类型、编码、服务器信息等）
        Server: Apache-Coyote/1.1 （Tomcat内置的HTTP连接器）
        Set-Cookie: JSESSIONID=123456
空行：（分隔响应头和响应体）
响应体：<html><body>Hello JavaWeb</body></html> （返回给客户端的内容）
③ 常用HTTP请求方法
方法 作用 特点
GET 获取资源（如查询数据） 参数拼在URL后、长度有限、不安全、可缓存
POST 提交数据（如新增/修改） 参数在请求体、长度无限制、相对安全、不可缓存
PUT 更新资源 幂等（多次请求结果一致）
DELETE 删除资源 幂等
④ 常用HTTP状态码
状态码分类 示例 含义
2xx（成功） 200 OK 请求成功，服务器返回数据
3xx（重定向） 302 Found 请求的资源已临时移动到新地址
4xx（客户端错误） 404 Not Found
400 Bad Request
403 Forbidden		 资源不存在
请求参数错误
无访问权限
5xx（服务器错误） 500 Internal Server Error
503 Service Unavailable		 服务器代码报错
服务器暂时不可用

1.2.3 URL & URI

  • URL（统一资源定位符）：完整地址，如 http://localhost:8080/myweb/user?name=admin
  • URI（统一资源标识符）：资源的唯一标识，如 /myweb/user（URL的核心部分）

1.2.4 会话（Session）与无状态

HTTP是「无状态」的，服务器无法识别连续请求是否来自同一客户端，因此需要：

  • Cookie：存储在客户端的小型文本文件，用于记录用户状态（如记住登录名）
  • Session：存储在服务器的用户会话数据，通过Cookie中的JSESSIONID关联

1.3 开发环境搭建（IDEA2025 + JDK24 + Tomcat11.0.11 + Windows11）

1.3.1 JDK24 安装与环境变量配置

步骤1：下载JDK24

下载地址：https://jdk.java.net/24/（选择Windows x64 Installer或压缩包）

步骤2：安装JDK24
  • 双击安装包，选择安装路径（建议：D:\Java\jdk-24，无中文/空格）
  • 取消「Set JAVA_HOME」自动配置（手动配置更稳定）
步骤3：配置Windows11环境变量
  1. 右键「此电脑」→「属性」→「高级系统设置」→「环境变量」
  2. 「系统变量」→「新建」：
    • 变量名：JAVA_HOME
    • 变量值：D:\Java\jdk-24（你的JDK安装路径）
  3. 编辑「Path」变量→「新建」：
    • %JAVA_HOME%\bin
    • %JAVA_HOME%\jre\bin（JDK24无需单独装JRE，自动包含）
  4. （可选）新建「CLASSPATH」：
    • 变量值：.;%JAVA_HOME%\lib\dt.jar;%JAVA_HOME%\lib\tools.jar
步骤4：验证安装
# 按下 Win + R → 输入 cmd → 回车 java -version # 正确输出示例： # java version "24" 2025-03-18 # Java(TM) SE Runtime Environment (build 24+36-2344) # Java HotSpot(TM) 64-Bit Server VM (build 24+36-2344, mixed mode, sharing) javac -version # 正确输出示例： # javac 24
❗ 注意：若提示「不是内部或外部命令」，检查： 1. JAVA_HOME路径是否正确； 2. Path中是否添加%JAVA_HOME%\bin； 3. 重启CMD窗口（环境变量生效需要重启）。

1.3.2 Tomcat11.0.11 安装与配置

步骤1：下载Tomcat11

下载地址：https://tomcat.apache.org/download-11.cgi

选择：64-bit Windows zip（压缩包版，无需安装，更灵活）

步骤2：解压与目录说明
  • 解压到无中文/空格路径，如：D:\Apache-Tomcat-11.0.11
  • 核心目录：
    • bin：启动/停止脚本（startup.bat、shutdown.bat）
    • conf：配置文件（server.xml、web.xml）
    • webapps：部署JavaWeb项目的目录
    • logs：日志文件（排查启动报错）
    • temp：临时文件
步骤3：配置Tomcat环境变量（可选，建议配置）
  1. 「系统变量」→「新建」：
    • 变量名：CATALINA_HOME
    • 变量值：D:\Apache-Tomcat-11.0.11（Tomcat解压路径）
  2. 编辑「Path」→「新建」：%CATALINA_HOME%\bin
步骤4：启动Tomcat11
# 方式1：双击 bin/startup.bat（会弹出黑窗口，关闭则Tomcat停止） # 方式2：CMD执行（推荐，可看日志） cd D:\Apache-Tomcat-11.0.11\bin startup.bat

启动成功验证：打开浏览器访问 http://localhost:8080，看到Tomcat默认首页（橙色小猫）即成功。

❗ Tomcat11启动常见问题： 1. 端口被占用：默认8080端口被占用 → 修改conf/server.xml中<Connector port="8080">为8081； 2. JDK版本不兼容：Tomcat11要求JDK17+，你的JDK24完全兼容； 3. 黑窗口闪退：打开CMD手动启动，查看报错日志（logs/catalina.out）。
步骤5：停止Tomcat
# 方式1：双击 bin/shutdown.bat # 方式2：CMD执行 cd D:\Apache-Tomcat-11.0.11\bin shutdown.bat

1.3.3 IDEA 2025.1.1 配置Tomcat11

步骤1：打开IDEA，配置JDK24
  1. IDEA → File → Project Structure → Platform Settings → SDKs
  2. 点击「+」→ 选择「Java」→ 选择JDK24安装路径（D:\Java\jdk-24）
  3. 确认「Java SDK version」显示24，点击Apply。
步骤2：添加Tomcat11服务器
  1. IDEA右上角 → 点击「Add Configuration...」
  2. 点击「+」→ 找到「Tomcat Server」→ 选择「Local」
  3. 在「Application server」右侧点击「Configure...」
  4. 选择Tomcat11解压路径（D:\Apache-Tomcat-11.0.11）→ 点击OK
  5. 确认「Name」为「Tomcat 11.0.11」，「HTTP port」为8080（和Tomcat配置一致），点击Apply。
💡 若IDEA中找不到Tomcat Server选项： 1. 检查是否安装Tomcat插件：File → Settings → Plugins → 搜索「Tomcat」→ 确保「Tomcat and TomEE Integration」已启用； 2. 你的IDEA已安装「com.poratu.idea.plugins.tomcat」插件，可直接识别Tomcat11。

1.3.4 部署第一个JavaWeb项目（验证配置）

步骤1：创建JavaWeb项目
  1. IDEA → File → New → Project
  2. 左侧选择「Java」→ 勾选「Web Application」→ 选择JDK24 → 点击Next
  3. 填写项目名称（如FirstJavaWeb）→ 选择项目路径 → 点击Finish。
步骤2：部署并运行
  1. 回到「Run/Debug Configurations」→ 选择Tomcat11 → 点击「Deployment」标签
  2. 点击「+」→ 选择「Artifact」→ 选择项目的war包（如FirstJavaWeb:war exploded）
  3. 「Application context」填写「/firstweb」（访问路径）→ 点击Apply
  4. 点击IDEA右上角「Run」按钮（绿色三角），IDEA会自动启动Tomcat并部署项目。
  5. 自动打开浏览器 → 访问 http://localhost:8080/firstweb，看到IDEA默认的index.jsp页面即成功。
