3.1 Servlet 基本认知

3.1.1 什么是 Servlet

Servlet 是运行在 服务器端（Tomcat） 的 Java 小程序，是 JavaWeb 处理 HTTP 请求、返回响应的核心组件。 简单说：浏览器发请求 → Tomcat 转发 → Servlet 处理 → 返回结果给浏览器。

3.1.2 版本关键说明（适配你的环境）

❗ 极其重要：Tomcat 10/11 包名变更
- Tomcat 9 及以前：javax.servlet.*
- Tomcat 10/11（你用的版本）：jakarta.servlet.*
这是 Servlet 6.0 规范，Maven依赖已适配，代码里必须用 jakarta 包，否则运行报错！

3.1.3 Servlet 作用

  • 接收浏览器（前端）发送的请求参数
  • 调用业务逻辑（Service 层、DAO 层）
  • 操作数据库（JDBC）
  • 返回响应数据（HTML、JSON、文本）
  • 实现页面跳转、登录验证、文件上传下载

3.2 Servlet 创建（Maven项目 + 注解版）

Servlet 6.0 推荐使用 注解 @WebServlet，无需配置 web.xml，Maven项目中直接创建即可。

3.2.1 IDEA 2025 创建 Servlet 步骤（Maven项目）

创建步骤
  1. 打开Maven JavaWeb项目 → 定位到 src/main/java
  2. 右键 → New → Package → 创建包：com.servlet（包名规范：域名反写）
  3. 右键包 → New → Java Class → 类名：HelloServlet
  4. 继承 HttpServlet（Maven已导入jakarta.servlet依赖，可直接导入）
  5. 添加注解 @WebServlet("/hello")
  6. 重写 doGet / doPost 方法

完整代码（Maven项目可直接复制运行）

package com.servlet;

// Maven依赖已导入，直接import jakarta包即可
import jakarta.servlet.ServletException;
import jakarta.servlet.annotation.WebServlet;
import jakarta.servlet.http.HttpServlet;
import jakarta.servlet.http.HttpServletRequest;
import jakarta.servlet.http.HttpServletResponse;
import java.io.IOException;
import java.io.PrintWriter;

// @WebServlet：访问路径 /hello
@WebServlet("/hello")
public class HelloServlet extends HttpServlet {

    // 处理 GET 请求
    @Override
    protected void doGet(HttpServletRequest request,
                         HttpServletResponse response)
            throws ServletException, IOException {

        // 1. 设置响应编码
        response.setContentType("text/html;charset=UTF-8");

        // 2. 获取输出流
        PrintWriter out = response.getWriter();

        // 3. 输出内容到浏览器
        out.write("<h1>Hello Servlet 6.0 + Tomcat 11 + Maven</h1>");
        out.write("<p>你的环境：JDK24 + IDEA2025 + Maven</p>");
    }

    // 处理 POST 请求
    @Override
    protected void doPost(HttpServletRequest request,
                          HttpServletResponse response)
            throws ServletException, IOException {
        // 让 POST 走 GET 逻辑，统一处理
        doGet(request, response);
    }
}

访问路径（Maven项目）

http://localhost:8080/javaweb-demo/hello

✔ javaweb-demo 是 pom.xml 中 <finalName> 配置的项目名；
✔ IDEA中配置Tomcat时，Deployment → Application context 需和 <finalName> 一致。

3.3 Servlet 生命周期（面试 + 核心原理）

Servlet 生命周期由 Tomcat 容器管理，我们只需要重写方法，不需要手动调用。

生命周期四阶段：
1. 加载（Class.forName）
2. 初始化 → init()（只执行 1 次）
3. 服务 → service() → doGet/doPost（每次请求都执行）
4. 销毁 → destroy()（服务器关闭时执行 1 次）

生命周期演示代码（Maven项目）

package com.servlet;

import jakarta.servlet.ServletConfig;
import jakarta.servlet.annotation.WebServlet;
import jakarta.servlet.http.HttpServlet;
import jakarta.servlet.http.HttpServletRequest;
import jakarta.servlet.http.HttpServletResponse;

@WebServlet("/life")
public class LifeServlet extends HttpServlet {

    // 1. 初始化：服务器启动 / 第一次访问时执行（只一次）
    @Override
    public void init(ServletConfig config) {
        System.out.println("Servlet 初始化完成...");
    }

    // 2. 服务：每次请求都会执行
    @Override
    protected void service(HttpServletRequest request,
                           HttpServletResponse response) {
        System.out.println("收到请求，执行 service...");
    }

    // 3. 销毁：服务器关闭时执行（只一次）
    @Override
    public void destroy() {
        System.out.println("Servlet 销毁...");
    }
}
✔ init()：只执行一次，说明 Servlet 是单例；
✔ service()：每次请求都会执行；
✔ destroy()：服务器正常关闭才执行（IDEA中点击Tomcat停止按钮）。

3.4 HttpServletRequest（请求对象）

HttpServletRequest 代表 浏览器发送给服务器的请求，用来获取：请求行、请求头、请求参数。

3.4.1 常用方法

// 1. 获取请求参数（最常用）
String username = request.getParameter("username");
String password = request.getParameter("password");

// 2. 获取多个同名参数（如爱好 checkbox）
String[] hobbies = request.getParameterValues("hobby");

// 3. 获取请求路径相关
String method = request.getMethod();       // GET/POST
String uri = request.getRequestURI();      // /javaweb-demo/hello
String ip = request.getRemoteAddr();       // 客户端IP

// 4. 解决 POST 请求中文乱码
request.setCharacterEncoding("UTF-8");

Servlet 接收参数（com.servlet.LoginServlet）

package com.servlet;

import jakarta.servlet.annotation.WebServlet;
import jakarta.servlet.http.HttpServlet;
import jakarta.servlet.http.HttpServletRequest;
import jakarta.servlet.http.HttpServletResponse;
import java.io.IOException;
import java.io.PrintWriter;

@WebServlet("/login")
public class LoginServlet extends HttpServlet {
    @Override
    protected void doPost(HttpServletRequest request,
                          HttpServletResponse response)
            throws IOException {

        // 解决 POST 中文乱码
        request.setCharacterEncoding("UTF-8");
        response.setContentType("text/html;charset=UTF-8");

        // 获取参数
        String username = request.getParameter("username");
        String password = request.getParameter("password");

        PrintWriter out = response.getWriter();
        out.write("用户名：" + username + "<br>");
        out.write("密码：" + password);
    }
}
❗ 乱码总结：
- POST：request.setCharacterEncoding("UTF-8")；
- GET：Tomcat 8+ 已自动解决，无需处理；
- 响应乱码：必须先写 response.setContentType("text/html;charset=UTF-8") 再获取输出流。

3.5 HttpServletResponse（响应对象）

HttpServletResponse 代表 服务器返回给浏览器的响应，用来输出内容、跳转页面。

3.5.1 常用方法

// 1. 设置响应编码（必须先写）
response.setContentType("text/html;charset=UTF-8");

// 2. 获取输出流（输出 HTML/文本）
PrintWriter out = response.getWriter();

// 3. 重定向（跳转）
response.sendRedirect("index.html"); // 跳转到webapp/index.html

// 4. 设置响应头
response.setHeader("Refresh","3;url=index.html"); // 3秒跳转

3.6 请求转发 & 重定向（高频面试 + 开发必懂）

对比项 请求转发 forward 重定向 redirect
写法 request.getRequestDispatcher("路径").forward(request,response); response.sendRedirect("路径");
浏览器地址栏 不变 改变
请求次数 1 次 2 次
数据共享 可通过 request 共享数据 不能共享
跳转范围 只能在服务器内部跳转 可跳外网（如百度）
Maven项目路径 转发到 WEB-INF 内文件：
request.getRequestDispatcher("/WEB-INF/index.jsp").forward(...)		 重定向不能访问 WEB-INF 内文件

使用场景

  • 查询、展示数据：用 请求转发（带数据）；
  • 新增、修改、删除、登录成功：用 重定向（避免重复提交）。

3.7 综合实战：登录验证（表单 + Servlet + 判断）

需求

用户名：admin 密码：123456 → 登录成功；否则失败。

LoginServlet 完整代码（Maven项目）

package com.servlet;

import jakarta.servlet.annotation.WebServlet;
import jakarta.servlet.http.HttpServlet;
import jakarta.servlet.http.HttpServletRequest;
import jakarta.servlet.http.HttpServletResponse;
import java.io.IOException;
import java.io.PrintWriter;

@WebServlet("/login")
public class LoginServlet extends HttpServlet {
    @Override
    protected void doPost(HttpServletRequest request,
                          HttpServletResponse response)
            throws IOException {

        // 编码
        request.setCharacterEncoding("UTF-8");
        response.setContentType("text/html;charset=UTF-8");

        // 接收参数
        String username = request.getParameter("username");
        String password = request.getParameter("password");

        PrintWriter out = response.getWriter();

        // 验证
        if ("admin".equals(username) && "123456".equals(password)) {
            out.write("<h2 style='color:green'>登录成功！欢迎你：" + username + "</h2>");
            // 3秒后跳转到首页
            response.setHeader("Refresh","3;url=/javaweb-demo/index.html");
        } else {
            out.write("<h2 style='color:red'>用户名或密码错误！</h2>");
            // 3秒后返回登录页
            response.setHeader("Refresh","3;url=/javaweb-demo/login.html");
        }
    }
}

3.8 Servlet 处理 AJAX 请求（核心实战 · 对应第二章AJAX）

3.8.1 AJAX 请求 vs 普通表单请求（核心区别）

第二章讲的前端AJAX请求和普通表单请求的核心差异：

对比项 普通表单请求 AJAX 异步请求
页面行为 整页刷新 局部刷新（不跳转、不刷新）
数据格式 默认 form-data / x-www-form-urlencoded 优先 JSON（前后端统一格式）
响应处理 服务器直接输出HTML到页面 服务器返回JSON数据，前端JS解析后渲染

3.8.2 核心准备：Servlet 返回 JSON 数据（关键配置）

AJAX请求要求后端返回JSON格式数据，需先导入JSON依赖（Maven），并配置响应格式：

步骤2：后端Servlet处理AJAX请求（核心）

文件路径：src/main/java/com/servlet/AjaxLoginServlet.java

package com.servlet;

import com.alibaba.fastjson2.JSON;
import com.alibaba.fastjson2.JSONObject;
import jakarta.servlet.ServletInputStream;
import jakarta.servlet.annotation.WebServlet;
import jakarta.servlet.http.HttpServlet;
import jakarta.servlet.http.HttpServletRequest;
import jakarta.servlet.http.HttpServletResponse;
import java.io.IOException;
import java.nio.charset.StandardCharsets;

@WebServlet("/ajaxLogin")
public class AjaxLoginServlet extends HttpServlet {

    @Override
    protected void doPost(HttpServletRequest request,
                          HttpServletResponse response)
            throws IOException {

        // 1. 核心配置：响应JSON格式 + 解决乱码（AJAX必配）
        response.setContentType("application/json;charset=UTF-8");
        response.setCharacterEncoding("UTF-8");

        // 2. 读取前端传递的JSON数据（关键：和普通表单参数不同）
        ServletInputStream inputStream = request.getInputStream();
        String jsonStr = new String(inputStream.readAllBytes(), StandardCharsets.UTF_8);
        
        // 3. 解析JSON为Java对象
        JSONObject jsonObject = JSON.parseObject(jsonStr);
        String username = jsonObject.getString("username");
        String password = jsonObject.getString("password");

        // 4. 业务逻辑：登录验证
        JSONObject result = new JSONObject(); // 要返回的JSON结果
        if ("admin".equals(username) && "123456".equals(password)) {
            result.put("success", true);
            result.put("msg", "欢迎你，" + username);
        } else {
            result.put("success", false);
            result.put("msg", "用户名或密码错误");
        }

        // 5. 返回JSON数据给前端
        response.getWriter().write(result.toJSONString());
    }
}

3.8.4 AJAX请求常见问题 & 解决方案

❗ 高频问题总结：
1. JSON解析失败：
- 前端必须设置 "Content-Type": "application/json"；
- 后端必须用 request.getInputStream() 读取（不能用 getParameter）；
2. 跨域问题（前后端分离时）：
新增CORS过滤器，放行跨域请求： 
package com.filter;

import jakarta.servlet.Filter;
import jakarta.servlet.FilterChain;
import jakarta.servlet.annotation.WebFilter;
import jakarta.servlet.http.HttpServletResponse;
import java.io.IOException;

@WebFilter("/*")
public class CorsFilter implements Filter {
    @Override
    public void doFilter(jakarta.servlet.ServletRequest request,
                         jakarta.servlet.ServletResponse response,
                         FilterChain chain) throws IOException, jakarta.servlet.ServletException {
        HttpServletResponse resp = (HttpServletResponse) response;
        // 放行所有域名（生产环境指定具体域名）
        resp.setHeader("Access-Control-Allow-Origin", "*");
        // 放行请求方法
        resp.setHeader("Access-Control-Allow-Methods", "GET,POST,PUT,DELETE");
        // 放行请求头
        resp.setHeader("Access-Control-Allow-Headers", "Content-Type");
        chain.doFilter(request, response);
    }
}
3. 乱码问题：
后端必须先设置 response.setContentType("application/json;charset=UTF-8") 再输出JSON；
4. 请求404：
检查Servlet的 @WebServlet 路径和前端fetch的URL是否一致（注意项目名 /javaweb-demo）。

3.8.5 简化版：表单格式AJAX请求（兼容传统方式）

如果前端不想传JSON，用传统表单格式，Servlet可简化：

// 前端修改（去掉JSON.stringify，改Content-Type）
const response = await fetch("/javaweb-demo/ajaxLogin", {
    method: "POST",
    headers: {
        "Content-Type": "application/x-www-form-urlencoded"
    },
    body: `username=${username}&password=${password}`
});

// 后端修改（直接用getParameter）
@Override
protected void doPost(HttpServletRequest request, HttpServletResponse response) throws IOException {
    response.setContentType("application/json;charset=UTF-8");
    // 直接获取表单参数（无需读流）
    String username = request.getParameter("username");
    String password = request.getParameter("password");
    
    // 后续逻辑和之前一致...
}

3.9 本章核心总结

  • ✔ Maven项目中Servlet 6.0依赖坐标：jakarta.servlet:jakarta.servlet-api:6.0.0，scope设为provided；
  • ✔ Tomcat11必须用 jakarta.servlet 包，Maven依赖已适配；
  • ✔ Servlet由注解 @WebServlet("/路径") 配置，无需web.xml；
  • ✔ 生命周期：init(1次) → service(每次) → destroy(1次)；
  • ✔ request：取参数、解决乱码、请求转发；response：输出内容、重定向；
  • ✔ 重定向：地址栏变、2次请求、不能共享数据；请求转发：地址栏不变、1次请求、可共享数据；
  • ✔ Servlet处理AJAX请求：优先返回JSON格式，JSON数据用request.getInputStream()读取，需配置CORS过滤器解决跨域。

第3章：Servlet 核心（小白完整版）

3.1 Servlet 是什么？工作原理

大白话：
Servlet 就是运行在服务器（Tomcat）上的 Java 小程序，专门用来：
• 接收浏览器发过来的请求
• 处理逻辑
• 给浏览器返回结果

工作流程（必懂）：

  1. 浏览器输入地址 → 发请求给 Tomcat
  2. Tomcat 根据地址找到对应的 Servlet
  3. Tomcat 调用 Servlet 的方法处理请求
  4. Servlet 把结果返回给 Tomcat
  5. Tomcat 把结果响应给浏览器

3.2 第一个 Servlet 程序

步骤：

  1. 创建 Maven Web 项目
  2. 导入 Servlet 依赖
  3. 写一个类，继承 HttpServlet
  4. 加 @WebServlet 注解配置访问路径
  5. 重写 doGet / doPost

pom.xml 依赖（Tomcat11 用 jakarta）

<dependency>
    <groupId>jakarta.servlet</groupId>
    <artifactId>jakarta.servlet-api</artifactId>
    <version>6.0.0</version>
    <scope>provided</scope>
</dependency>

HelloServlet.java

package com.servlet;

import jakarta.servlet.annotation.WebServlet;
import jakarta.servlet.http.HttpServlet;
import jakarta.servlet.http.HttpServletRequest;
import jakarta.servlet.http.HttpServletResponse;
import java.io.IOException;
import java.io.PrintWriter;

@WebServlet("/hello")
public class HelloServlet extends HttpServlet {
    @Override
    protected void doGet(HttpServletRequest request, HttpServletResponse response) throws IOException {
        response.setContentType("text/html;charset=UTF-8");
        PrintWriter out = response.getWriter();
        out.write("<h1>Hello Servlet！你好</h1>");
    }
}

访问：http://localhost:8080/项目名/hello

3.3 Servlet 生命周期（4 步）

  1. 加载：第一次访问时，加载类
  2. 创建：执行构造方法，只执行 1 次
  3. 初始化：执行 init()，只 1 次
  4. 服务：执行 service() → 调用 doGet/doPost，每次访问都执行
  5. 销毁：服务器关闭时执行 destroy()
💡 一句话记住：
单例、线程不安全；
创建初始化一次，服务多次，销毁一次。

3.4 请求与响应（HttpServletRequest / HttpServletResponse）

HttpServletRequest（请求对象）

  • 获取前端参数：request.getParameter("name")
  • 获取请求方式：request.getMethod()
  • 存数据（一次请求内有效）：setAttribute / getAttribute
  • 设置请求编码：request.setCharacterEncoding("UTF-8")

HttpServletResponse（响应对象）

  • 设置响应类型+编码：setContentType("text/html;charset=UTF-8")
  • 获取输出流：getWriter()
  • 重定向：sendRedirect("地址")

3.5 请求转发 vs 重定向（面试+开发必考）

1）请求转发：request.getRequestDispatcher("地址").forward(request,response)

  • 地址栏不变
  • 一次请求
  • 可以共享 request 数据
  • 只能跳站内地址

2）重定向：response.sendRedirect("地址")

  • 地址栏变
  • 两次请求
  • 不能共享 request 数据
  • 可以跳站外

3.6 Filter 过滤器（网站门卫 · 最常用）

Filter 也是 Servlet 规范的一部分！
作用：请求到达 Servlet 之前，先经过 Filter 统一处理。

最常用 2 个：

  • 全局编码过滤器（解决全站乱码）
  • 登录拦截过滤器（没登录不让进）

开发步骤（固定）

  1. 实现 jakarta.servlet.Filter 接口
  2. 重写 doFilter()
  3. 加 @WebFilter("拦截路径")
  4. 必须调用 chain.doFilter(request,response) 放行

实战1：全局编码过滤器（必用）

package com.filter;

import jakarta.servlet.Filter;
import jakarta.servlet.FilterChain;
import jakarta.servlet.ServletException;
import jakarta.servlet.annotation.WebFilter;
import jakarta.servlet.http.HttpServletRequest;
import jakarta.servlet.http.HttpServletResponse;
import java.io.IOException;

@WebFilter("/*")
public class EncodingFilter implements Filter {
    @Override
    public void doFilter(jakarta.servlet.ServletRequest request,
                         jakarta.servlet.ServletResponse response,
                         FilterChain chain) throws IOException, ServletException {
        HttpServletRequest req = (HttpServletRequest) request;
        HttpServletResponse resp = (HttpServletResponse) response;

        req.setCharacterEncoding("UTF-8");
        resp.setContentType("text/html;charset=UTF-8");
        resp.setCharacterEncoding("UTF-8");

        chain.doFilter(request, response);
    }
}

实战2：登录拦截过滤器

package com.filter;

import jakarta.servlet.Filter;
import jakarta.servlet.FilterChain;
import jakarta.servlet.ServletException;
import jakarta.servlet.annotation.WebFilter;
import jakarta.servlet.http.HttpServletRequest;
import jakarta.servlet.http.HttpServletResponse;
import jakarta.servlet.http.HttpSession;
import java.io.IOException;

@WebFilter("/*")
public class LoginFilter implements Filter {
    @Override
    public void doFilter(jakarta.servlet.ServletRequest request,
                         jakarta.servlet.ServletResponse response,
                         FilterChain chain) throws IOException, ServletException {
        HttpServletRequest req = (HttpServletRequest) request;
        HttpServletResponse resp = (HttpServletResponse) response;
        String uri = req.getRequestURI();

        // 放行登录页、登录接口、静态资源
        if(uri.contains("login") || uri.contains(".css") || uri.contains(".js")){
            chain.doFilter(request,response);
            return;
        }

        HttpSession session = req.getSession();
        Object user = session.getAttribute("loginUser");
        if(user != null){
            chain.doFilter(request,response);
        }else{
            resp.sendRedirect(req.getContextPath()+"/login.html");
        }
    }
}

3.7 Listener 监听器（服务器监控 · 常用）

Listener 也是 Servlet 规范！
作用：监听 项目启动/关闭、Session 创建/销毁 等事件，自动执行代码。

只学最常用 2 个：

  • ServletContextListener：监听项目启动/关闭
  • HttpSessionListener：监听会话，统计在线人数

开发步骤

  1. 实现对应监听器接口
  2. 加 @WebListener 注解

实战：统计在线人数

package com.listener;

import jakarta.servlet.ServletContext;
import jakarta.servlet.ServletContextEvent;
import jakarta.servlet.ServletContextListener;
import jakarta.servlet.annotation.WebListener;
import jakarta.servlet.http.HttpSessionEvent;
import jakarta.servlet.http.HttpSessionListener;

@WebListener
public class OnlineListener implements ServletContextListener, HttpSessionListener {

    private ServletContext app;

    @Override
    public void contextInitialized(ServletContextEvent sce) {
        app = sce.getServletContext();
        app.setAttribute("online", 0);
    }

    @Override
    public void sessionCreated(HttpSessionEvent se) {
        int count = (Integer) app.getAttribute("online");
        app.setAttribute("online", count + 1);
    }

    @Override
    public void sessionDestroyed(HttpSessionEvent se) {
        int count = (Integer) app.getAttribute("online");
        app.setAttribute("online", count - 1);
    }
}

页面展示：当前在线：${applicationScope.online}

3.8 Servlet 处理 AJAX 请求（小白版 · 对应第二章）

核心一句话

AJAX就是前端不刷新页面发请求，后端返回JSON数据，前端用JS渲染结果。

小白版实战（不用JSON，简化版）

1）前端代码（webapp/simple-ajax.html）

<!DOCTYPE html>
<html>
<body>
    用户名：<input id="uname"><br>
    <button onclick="ajaxTest()">测试AJAX</button>
    <div id="result"></div>

    <script>
        function ajaxTest() {
            // 1. 创建AJAX对象
            let xhr = new XMLHttpRequest();
            // 2. 配置请求
            xhr.open("POST", "/javaweb-demo/simpleAjax");
            xhr.setRequestHeader("Content-Type", "application/x-www-form-urlencoded");
            // 3. 发送请求
            xhr.send("username=" + document.getElementById("uname").value);
            // 4. 接收响应
            xhr.onreadystatechange = function() {
                if (xhr.readyState == 4 && xhr.status == 200) {
                    // 显示结果（局部刷新）
                    document.getElementById("result").innerText = xhr.responseText;
                }
            }
        }
    </script>
</body>
</html>

2）后端Servlet（com.servlet.SimpleAjaxServlet）

package com.servlet;

import jakarta.servlet.annotation.WebServlet;
import jakarta.servlet.http.HttpServlet;
import jakarta.servlet.http.HttpServletRequest;
import jakarta.servlet.http.HttpServletResponse;
import java.io.IOException;

@WebServlet("/simpleAjax")
public class SimpleAjaxServlet extends HttpServlet {
    @Override
    protected void doPost(HttpServletRequest request, HttpServletResponse response) throws IOException {
        request.setCharacterEncoding("UTF-8");
        response.setContentType("text/plain;charset=UTF-8");
        
        String username = request.getParameter("username");
        response.getWriter().write("你好，" + username + "！这是AJAX返回的结果");
    }
}
💡 小白重点：
1. AJAX不用刷新页面，点按钮就能显示结果；
2. 后端返回普通文本也可以，不用非要JSON；
3. 路径一定要写对：/项目名/注解路径。
