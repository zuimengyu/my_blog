1. Git全局配置

配置用户信息（所有仓库生效，仅需配置一次）

# 配置用户名
git config --global user.name "你的名字"
# 配置邮箱
git config --global user.email "你的邮箱"
# 查看配置
git config --list
# 查看Git版本
git --version

2. 核心基础命令

仓库初始化

# 初始化本地仓库
git init
# 克隆远程仓库
git clone 远程仓库地址

代码提交

# 添加单个文件到暂存区
git add 文件名
# 添加所有变更文件
git add .
#取消添加（从暂存区移除）
git restore --staged .
# 提交到本地仓库
git commit -m "提交说明"
# 查看工作区状态
git status
# 查看提交历史
git log

分支操作

# 查看所有分支
git branch
# 创建分支
git branch 分支名
# 切换分支
git checkout 分支名
# 创建并切换分支
git checkout -b 分支名
# 合并分支到当前分支
git merge 分支名
# 删除本地分支
git branch -d 分支名

远程交互

# 查看远程仓库
git remote -v
# 关联远程仓库
git remote add origin 远程地址
# 推送并关联远程分支
git push -u origin 分支名
# 拉取远程更新（拉取+合并）
git pull
# 仅拉取不合并
git fetch

撤销与回退

# 丢弃工作区修改
git checkout -- 文件名
# 取消暂存
git reset HEAD 文件名
# 回退到指定版本
git reset --hard 版本号
# 打版本标签
git tag v1.0

3. Git核心三层架构

Git的核心存储模型，理解后才能熟练使用

  • 工作区：本地电脑的文件目录，直接编辑代码
  • 暂存区：临时存放待提交的变更，git add写入
  • 版本库：.git文件夹，存储所有历史版本与元数据

4. Git数据对象

  • Blob：存储文件的原始数据
  • Tree：存储目录结构+文件指针
  • Commit：版本快照，包含树、父提交、作者信息
  • Tag：版本标记，用于发布稳定版

5. Git命令体系分类

  • 配置类：config
  • 仓库创建：init、clone
  • 快照操作：add、commit、status、diff
  • 分支合并：branch、checkout、merge、rebase
  • 远程交互：remote、fetch、pull、push
  • 撤销修复：reset、revert、checkout --

6. Git扩展模块

  • Git Hooks：钩子脚本（提交前、推送后自动执行）
  • Submodules：子仓库管理
  • Reflog：恢复误删分支

7. 企业标准分支模型

main（主分支）：最终上线版本，永远稳定，禁止直接修改
↓
dev（开发分支）：功能汇总分支，所有功能完成后合并到此测试
↙ ↓ ← ↓ ↘
feature-login（A开发登录）
feature-register（B开发注册）
feature-home（C开发主页面）
feature-star（D开发收藏）
feature-setting（E开发设置）
✅ 关键：不是复制粘贴合并，Git会自动合并代码；每个功能一个分支，不是多个项目文件夹！

8. 团队协作完整实战步骤

Git 团队协作完整实战教学文档

✅ 文档说明：完全保留原始命令，无任何删减，严格按照项目负责人A、成员B、成员C的角色分阶段执行，适用于Gitee/GitHub平台团队协作

第一阶段：项目初始化 + 团队协作全流程

👨‍💻 角色：项目负责人 A

# 1. 配置全局用户信息（A/B/C统一配置）
git config --global user.name "项目负责人A"
git config --global user.email "a@xxx.com"

git config --global user.name "成员B"
git config --global user.email "b@xxx.com"

git config --global user.name "成员C"
git config --global user.email "c@xxx.com"

# 2. 初始化本地Git仓库
git init

# 3. 添加README文件到暂存区
git add README.md

# 4. 提交到本地仓库
git commit -m "A创建项目，新建README.md"

# 5. 查看当前分支
git branch

# 6. 创建并切换到dev开发分支
git checkout -b dev

# 7. 再次查看分支（确认dev分支创建成功）
git branch

# 8. 关联远程Gitee仓库
git remote add origin https://gitee.com/your-name/my-project.git

# 9. 推送main主分支到远程，并建立关联
git push -u origin main

# 10. 推送dev开发分支到远程，并建立关联
git push -u origin dev

👨‍💻 角色：成员 B

# 1. 克隆远程仓库到本地
git clone https://gitee.com/your-name/my-project.git

# 2. 查看所有分支（本地+远程）
git branch -a

# 3. 切换到dev开发分支
git checkout dev

# 4. 添加所有修改文件到暂存区
git add .

# 5. 提交修改到本地仓库
git commit -m "B修改README，添加功能模块"

# 6. 推送修改到远程dev分支
git push origin dev

👨‍💻 角色：成员 C

# 1. 克隆远程仓库到本地
git clone https://gitee.com/your-name/my-project.git

# 2. 切换到dev开发分支
git checkout dev

# 3. 添加所有修改文件到暂存区
git add .

# 4. 提交修改到本地仓库
git commit -m "C新增Function.java文件"

# 5. 推送修改到远程dev分支
git push origin dev

👨‍💻 角色：项目负责人 A（合并代码）

# 1. 切换到dev分支
git checkout dev

# 2. 拉取远程dev分支最新代码
git pull origin dev

# 3. 切换到main主分支
git checkout main

# 4. 将dev分支代码合并到main主分支
git merge dev

# 5. 拉取远程main分支最新代码（同步远程）
git pull origin main

后续修改：迭代开发 + 最终上线流程

👨‍💻 角色：项目负责人 A / 全体成员

# 1. 切换到dev开发分支
git checkout dev

# 2. 拉取远程dev分支最新代码
git pull origin dev

# 3. 添加修改文件到暂存区
git add .

# 4. 提交修改
git commit -m "修改了xxx"

# 5. 推送修改到远程dev分支
git push origin dev

# 6. 切换到main主分支
git checkout main

# 7. 合并dev分支代码到main
git merge dev

# 8. 推送最终代码到远程main分支（上线）
git push origin main
📌 核心总结：
1. 主分支main：仅用于上线，保持稳定
2. 开发分支dev：团队日常开发、合并代码
3. 所有修改先提交dev，测试无误后合并main

9. 常见疑问解答

Q1：合并是手动复制文件夹吗？

❌ 不是！Git会自动合并代码，仅冲突时需要手动修改，无需复制粘贴。

Q2：多人开发是共用一个分支还是多个分支？

✅ 一人/一个功能一个feature分支，完成后合并到dev，最后合并到main。

Q3：多个分支会有多个项目文件夹吗？

❌ 只有一个项目文件夹，分支是Git内部的版本管理，切换分支自动切换代码。

Q4：dev怎么合并到main？

✅ 用git merge dev命令，Git自动合并，不是删除main替换。
